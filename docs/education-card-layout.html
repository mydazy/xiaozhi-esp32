03 · 启蒙触发规则（核心）

什么时候激活，什么时候跳过

教育卡是"机会教学"，不是"必显示"。命中 → 大方激活；超纲 → 静默跳过，保持 emoji/对话画面，不打扰。

激活校验函数

bool should_activate_card(const char* left, const char* right) {
    if (is_pinyin(left)) {
        // PY-mode: 主秀汉字必须在 3000 小学 + ≤ 4 字
        int n = utf8_char_count(right);
        return n >= 1 && n <= 4 && all_in_3000(right);
    } else {
        // EN-mode: 56 → 48 兜底 → 跳过
        // ≤ 10 走 56，11-12 走 48，> 12 直接跳过
        int n = utf8_char_count(left);
        if (n < 1 || n > 12) return false;
        // 48 实测兜底（48 装不下也跳过）
        return lv_text_get_width(left, strlen(left), &font_48_lat, 0, 0) <= 280;
    }
}

ShowEduCard("letter", "Aa", nullptr, example_chinese) + UpdateEduExplain(example_words)。

为什么不另立 80-100 px 超大字母档

  • 资源限制：80 px 1bpp 即使仅 26 字母也要 ~50 KB；100 px 要 ~80 KB。两个档 + 字母 + 例词中文 = ~150 KB 增量。
  • 视觉一致性：v8 已锁 56 主秀。新字号会破坏整体节奏（用户看 56 → 80 → 56 觉得突兀）。
  • 56 已经够大气：单字 "A" 在 56 px 下约 35 px 宽，加大写小写并排约 65 px = 占屏 23%，配 OLED 黑底 + 大留白，启蒙感足够。
  • 跑马灯增强：底部 3-5 个 A 开头单词跑过去，比"超大单字 A"教学价值更高（字母 ↔ 单词关联）。

LVGL 端实现

void mcp_show_word_card(const char* word, const char* chinese,
                       const char* syllables, const char* story_ctx,
                       int duration_ms) {
    if (!should_activate_card(word, chinese)) return;

    render_main(word, &font_56_lat, POS_EN_STORY);  // top: 80（让出顶位给 Phonics）
    render_sub_below_main(chinese, &font_20_cn);    // 紧贴主行下方

    if (syllables) render_top(syllables, &font_30_lat);  // Phonics 音节拆分
    if (story_ctx) render_explain_marquee(story_ctx);

    schedule_hide_card(duration_ms);                // 4 秒后自动隐藏
}

为什么这个设计学英语效率最高

  • 听 + 读 同步：TTS 朗读 cat 时屏幕显示 cat，孩子的"听觉记忆"和"视觉记忆"同时建立。
  • 情境记忆：单字不孤立，活在 "The little cat jumped" 的句子里——剑桥研究证明情境记忆效率比孤立词高 3-5 倍。
  • 中文桥梁不喧宾夺主：副位 20 px 让"猫"是辅助而非焦点，孩子的注意力锁定 56 px 的 "cat"。
  • 4 秒后自动消失：不打断故事节奏，跟亲子共读时妈妈指着图说 "this is a cat" 的体感一致。
  • 同词重复触发：同一个 cat 在不同故事中可能被触发 5-7 次，达到记忆牢固阈值。
