1 概览

产品定位

3-9 岁启蒙：LLM TTS 流抽出关键词时显示教育卡。统一两行布局：顶部辅助（30 px 薄荷绿）+ 主秀大字（56 px 金黄）· 仅 2 类：hanzi（汉字）+ word（英文单词）。

2 字体规范

教育卡用 3 档字号 · cbin 压缩 · 总 Flash ~2.4 MB · 88px 字体保留作时钟主屏（不参与教育卡）。

位置字号字重字符集颜色Flash

主秀兜底（EN 11-12 字符） 48 px 700 Bold ASCII + 拼音（无 CJK） #FFCA28 金 ~77 KB
顶部辅助 · 拼音 / IPA 30 px 600 SemiBold ASCII + 拼音 + IPA + 数学 + 仅 ~30 启蒙 CJK #A5D6A7 ~79 KB

5 触发链路（2 入口 · 单卡架构）

教育卡有 2 个触发入口，最终都走 ShowEduCard(main, top) 渲染。单卡架构 · 无 queue · 无 timer · 无 category。

负面案例（不触发 / 跳过）

场景错误示例原因
古诗朗诵 "床前明月光，[静夜思_jìng yè sī]" 古诗场景靠 TTS 即可，不显示卡（信息密度过高）

超 12 字符英文 "学会 [communication_沟通]" 13 字符超上限，端侧跳过
拼音无声调 "看 [好_hao]" 3-9 岁靠声调记发音，无声调不教学
单句多张卡 "看 [书包_shū bāo][苹果_apple]" 端侧只取第一张 · 单卡架构（音画同步）
用错分隔符 "看 [书包|shū bāo]" 必须用 _ 下划线（| 视觉接近字母 l/I）
触发频率排序：指物问名 ≈ 生字问询 ≈ 故事生词（合计 80%+ 高频）· 英文 + 绘本 = 15% · 其他场景 5%。详见 edu-prompt-v6.md。

入口 ① · MCP show_card（云端 LLM 主动触发）

// boards/common/education_mcp_tools.cc
mcp.AddTool("self.education.show_card",
    "显示教育卡片(两行布局)...",
    PropertyList({
        Property("main", kPropertyTypeString),
        Property("top",  kPropertyTypeString, ""),
    }),
    [ui](const PropertyList& p) -> ReturnValue {
        std::string main_t = p["main"].value<std::string>();
        std::string top    = p["top"].value<std::string>();
        if (main_t.empty()) return std::string("main 不能为空");
        Application::Schedule([ui, main_t, top]() {
            ui->ShowEduCard(main_t.c_str(), top.c_str());
        });
        return std::string("OK");
    });

入口 ② · 聊天文本正则（LLM TTS 流被动抽词）

LLM 在 TTS sentence 文本中嵌入 [main_top] 标记，端侧正则提取 + 渲染。

格式例maintop
两段 [main_top][好_hǎo]好hǎo
两段[书包_shū bāo]书包shū bāo
两段[apple_苹果]apple苹果
多于一个 _[a_b_c]⛔ 解析拒绝
缺 _[apple]⛔ 解析拒绝
分隔符：使用 _ 下划线(不是 | 竖线)。| 视觉接近字母 l/I · LLM 经常漏写；_ 不会跟字母混淆。

端侧解析流程(application.cc · 单卡架构)

// 1. 提取教育卡(严格单卡 · 音画同步)
std::optional<EduCard> ExtractEduCard(const std::string& s) {
    auto lp = s.find('[');
    if (lp == std::string::npos) return std::nullopt;
    auto rp = s.find(']', lp + 1);
    if (rp == std::string::npos) return std::nullopt;
    EduCard hit;
    if (!ParseEduCard(s, lp, rp, &hit)) return std::nullopt;
    return hit;
}

// 2. 投递到 main loop(state + FontGIF 守护)
void TriggerEduCard(EduCard hit) {
    Application::GetInstance().Schedule([hit = std::move(hit)]() {
        if (state != kDeviceStateSpeaking) return;
        if (!ui || ui->IsFontGifActive()) return;
        ui->ShowEduCard(hit.main.c_str(), hit.top.c_str());
    });
}

6 激活校验(PickFont)

非所有输入都激活：超纲 / 超长 / 字体缺失 → 静默跳过，保持当前画面，不打断对话。

决策树(2 参数版本 · 简化后)

const lv_font_t* PickFont(const char* main_text, bool is_cjk,
                          const lv_font_t* font_56,
                          const lv_font_t* font_48) {
    if (!main_text || !main_text[0] || !font_56) return nullptr;
    int n = Utf8CharCount(main_text);
    if (n < 1) return nullptr;

    if (is_cjk) {                          // 汉字
        return n <= 4 ? font_56 : nullptr;
    }
    // 英文
    if (n > 12) return nullptr;
    lv_point_t sz;
    lv_text_get_size(&sz, main_text, font_56, ...);
    if (sz.x <= 280) return font_56;     // 280 安全宽

    if (!font_48) return nullptr;
    lv_text_get_size(&sz, main_text, font_48, ...);
    return sz.x <= 280 ? font_48 : nullptr;
}

3-9 岁产品逻辑：
  • 视觉聚焦 · 主秀大字永远在屏幕中央偏下，孩子第一眼看到 → 学习重点
  • 辅助前置 · 辅助文字永远在顶部，理解上下文（拼音读音 / 中文意思 / 类别提示）
  • 节奏一致 · 卡片之间切换视觉位置不变，只有内容变 → 孩子不被布局变化分心
  • 降低信息密度 · 删 bottom 第三行 · 适合 3-9 岁注意力广度

# 规则
- 每句最多 1 张卡（音画同步 · 严格单卡）
- 超过上限 / 拼音无声调 自动跳过，不打断对话
- 优先使用入口 ① 嵌入式（自然 · 不卡顿 TTS）
- 入口 ② MCP 适合"用户主动问"场景

产品决策（设计原则）

  • 严格单卡 · 每句 LLM 输出最多 1 张卡（音画同步 · 无链式弹卡）
  • 单一两行布局 · 顶部辅助 + 主秀大字 · y=60/110 锚点固定 · 不分类
  • 跳过不报错 · 超纲 / 超长 / 字体缺失 → 静默跳过 · 不打断对话
  • 同轮屏蔽 FontGIF · GIF 显示中 ShowEduCard 直接 return · 下轮 Speaking 清场
  • 仅 2 类内容 · 汉字组词(≤4 字) + 英文单词(≤12 字符) · 删除 letter/phonics/math/poem/topic/color/pinyin 7 类
  • 3-9 岁视觉一致 · 不论场景视觉位置不变，孩子聚焦内容不被布局分心
  • YAGNI · API 不分类 · 协议 2 字段 · 极简到不能再简
