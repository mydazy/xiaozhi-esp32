4 HTTP binary mode（Patch B）

Patch B 把 HTTP body 从 HEX 编码（× 2 流量 + CPU 重）改为 binary 直读模式。配合 binary mode 引用计数 + 白名单 + body_consumed 偏移。

4.1 HTTP 流程

步命令说明
1AT+MHTTPCREATE创建 HTTP session
2AT+MHTTPCFG="ssl",...SSL 配置
3AT+MHTTPCFG="encoding",1,0尝试 binary · 失败回退 HEX
4AT+MHTTPREQUEST发起请求
5URC +MHTTPURC:"header"/"content"/"err"/"ind"异步通知
6Read · body_consumed_ 偏移16 KB 阈值才 erase（防抖动）
7Close · 50ms 排空残留 binary URC退出 binary mode

5 优雅关机

为什么必须 AT+CFUN=4 而非直接断电：DTR 引脚未接线（GPIO_NUM_NC），无法用硬件信号通知模组进入 sleep。直接断 LDO 会让 PPP/RRC 在服务端侧悬挂，下次注册被运营商基站短暂拒绝。CFUN=4 给模组 800ms 主动释放窗口。

6.3 binary mode 期间的状态盲点

MP3 长时间播放期间（binary mode ref > 0 持续）AT+CEREG? 和 AT+COPS? 不被白名单允许 · GetBoardJson 无法刷新运营商 / 注册状态 · OTA 上报的网络信息可能滞后到 binary mode 退出后才更新。评审 P1 建议加入白名单。
