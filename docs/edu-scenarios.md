## 三、被动学习场景（讲故事/聊天嵌入式教学）⭐

### 3.0 启蒙策略：自然拼读优先（不用 IPA 音标）

**3-10 岁中国孩子缺英语环境** → 自然拼读 (Phonics) 比 IPA 音标更易学、易记：

| 维度 | 自然拼读 | IPA 音标 |
|---|---|---|
| 学习曲线 | 字母组合规则（趣味韵律） | 48 个新符号需记忆 |
| 趣味性 | ✅ chant/song 教学 | ❌ 符号枯燥 |
| 中国适用 | ✅ 配 AI 高频朗读弥补环境短板 | ⚠️ 传统教学但孤立 |
| 卡片视觉 | `c-a-t` / `ban-an-a` 字母分段 | `/kæt/` 特殊符号 |
| 启蒙年龄 | 3-9 岁主线 | 9 岁+ 查词典再学 |

**当前策略**：
- ✅ 卡片 top 字段统一用自然拼读（`c-a-t` / `ap-ple` / `ban-an-a`）
- ❌ 不输出 IPA 国际音标（设备端字库虽含 IPA，但启蒙阶段不展示）
- ⏭️ 真机测试排版后再评估是否需要加 IPA 行（可能 4 行布局：拼读 + main + IPA + 释义）

### 3.1 设计目标

孩子主动求知场景频次有限。**真正高频的是被动场景**：
- 讲故事时遇到关键词 → 自然弹卡（不打断故事节奏）
- 聊天对话提到事物 → 顺势显示
- 启蒙类陪伴对话 → 多模态强化记忆

**关键约束**：
1. **TTS 朗读自然**——不能读出格式标记符号
2. **协议极简**——LLM prompt 只能简单约束，复杂标签易被截断
3. **设备端自解析**——纯字符串扫描，不依赖服务端中间件
4. **故事英文天然分隔**——中文 + 英文混排时，ASCII 字符段即可定位

### 3.2 协议：括号双语模式（设备端正则）

**LLM 输出**（自然中英混排）：
```
小红帽看到一只 apple（苹果），咬了一口。后来遇见了 wolf（狼）。
```

**TTS 朗读**（多数 TTS 引擎对全角括号轻停顿，双语朗读）：
> 小红帽看到一只 apple ... 苹果，咬了一口。后来遇见了 wolf ... 狼。

→ **双语对照朗读，符合启蒙"先英后中"教学法**。

**设备端处理**（`main/application.cc::ExtractEduWordCard`）：
1. 监听 TTS `sentence_start` 事件文本
2. 纯字符串扫描：找全角 `（`/`）` + 向左扫 ASCII 字母
3. 提取 `word` + `desc`，调 `ShowEduCard` 弹卡
4. 失败 return false，不影响主流程

### 3.3 三种格式兼容矩阵（同一解析器）

| LLM 输出 | 解析结果 | 卡片行数 |
|---|---|---|
| `apple（苹果）` | word=apple · bottom=苹果 · top="" | **2 行**（main + bottom）|
| `apple（苹果，ap-ple）` | word=apple · bottom=苹果 · top=ap-ple | **3 行** |
| `apple（苹果，/ˈæp.əl/）` | word=apple · bottom=苹果 · top=IPA | 3 行（兼容 IPA）|

`BuildEduCard` 内 `has_top = top[0] != '\0'` 自动判断，2/3 行布局自适应。

### 3.4 LLM Prompt 工程（双场景分工 ⭐）

#### A. 故事/聊天（被动 · 极简 2 行卡）

```
讲故事/聊天遇到适合 3-10 岁孩子学的英文词，按这种格式：

  英文（中文）

约束：一句最多 1 词 / 中文 ≤7 字 / 不写拼读音标 / 单词 2-16 字母
```

#### B. 主动问答（用户求知 · 完整 3 行卡）

走 MCP tool `show_card(word, banana, ban-an-a, 香蕉)`，含自然拼读。

| 场景 | 输出 | 卡片 | 字段 |
|---|---|---|---|
| **被动渗透** | `apple（苹果）` 括号 | 2 行 | main + bottom |
| **主动问答** | MCP tool call | 3 行 | top + main + bottom |

### 3.5 真实示例（讲故事）

**LLM 输出**：
```
今天天气很好，小明带着小狗 dog（小狗）去公园玩。
路上遇到一只 cat（猫咪），小狗追着 cat 跑了好远。
公园里有一棵高高的 tree（大树），停着一只 bird（小鸟）。
```

**屏幕同步弹卡**（4 张 2 行简洁卡）：
1. dog | 小狗
2. cat | 猫咪
3. tree | 大树
4. bird | 小鸟

→ 听一遍故事 = 学 4 个英文单词 + 中文双向强化。

### 3.6 边界保护

| 场景 | 行为 |
|---|---|
| 流式切片 `apple（苹` | 找不到 `）`，return false |
| 半角 `(apple)` | 不匹配全角，return false |
| 单词 <2 / >16 字母 | 长度过滤 |
| 多词一句 | 取第一个匹配（避免视觉跳动） |
| MCP + 正则并发 | 后到覆盖前面（async 删除无 race）|

## 十、协议设计 FAQ

**Q1：括号内为什么不带额外格式？**
A：LLM prompt 越简单越稳定。`英文（中文）` 是 LLM 训练数据中天然存在的格式，零指令成本就能高质量输出。

**Q2：故事中可以插多少标记？**
A：建议每段 2-3 个，每句最多 1 个（避免视觉切卡过快）。设备端取首个匹配防过载。

**Q3：服务端要改吗？**
A：**不需要**。括号格式是 LLM 自然中英混排，服务端零中间件，设备端正则直接处理。

**Q4：流式 TTS 切片到 `apple（苹` 怎么办？**
A：设备端 `ExtractEduWordCard` 找不到 `）` 时返回 false 不弹卡。等 `sentence_start` 完整下发后再处理。

**Q5：能不能扩展 hanzi/pinyin 也走括号？**
A：**不能**——hanzi 是 CJK 字符，没有 ASCII 天然分隔，要解析就得加复杂标签或反查字库。hanzi/pinyin 走主动 MCP 工具，**职责清晰**。

**Q6：括号内可以多种格式吗？**
A：可以。设备端兼容 `apple（苹果）` / `apple（苹果，ap-ple）` / `apple（苹果，/IPA/）`。LLM 想给信息就给，想简洁就给两段。

## 十一、⭐ 摇一摇互动启蒙（核心创新场景）

### 11.1 架构：设备端纯触发器 + 云端智能推荐

**职责清晰分离**：
- **设备端**：硬件感知（加速度计）+ 状态判断（仅 Idle 接管）+ 触发消息上报
- **云端**：用户档案管理 + 艾宾浩斯曲线计算 + 个性化场景推荐 + LLM 朗读

数据流：

```
摇一摇（硬件）
  ↓ SC7A20H 加速度计三轴模长突变检测
  ↓ shake_detect_task_ → callback
设备端
  ↓ 检查 GetDeviceState() == kDeviceStateIdle（防干扰对话/播放/QR/EduCard）
  ↓ PlaySound(OGG_POPUP) 立即提示音反馈
  ↓ SendTextToAI("[摇一摇启蒙]")  // 通过现有 LLM 通道发触发指令
云端服务（java_mydazy）
  ↓ LLM system prompt 识别 [摇一摇启蒙] 前缀
  ↓ 查用户档案：年龄 / 已学词 / 上次复习时间戳
  ↓ 艾宾浩斯曲线计算：哪些词该复习了
  ↓ 推荐策略：
      - 70% 出新场景（按年龄+主题轮换）
      - 30% 出复习卡（已学词）
  ↓ 调用 mcp.tools.show_card 推送到设备
  ↓ TTS 流朗读个性化造句
设备端接收
  ↓ ShowEduCard 弹卡 + TTS 同步播放
```

### 11.2 设备端实现（已接入）

`mydazy_p30_4g_board.cc::ShakeDetectTaskEntry` 检测到 shake 后：

```cpp
Application::GetInstance().Schedule([]() {
    auto& app = Application::GetInstance();
    if (app.GetDeviceState() != kDeviceStateIdle) {
        ESP_LOGD(TAG, "Shake ignored: not idle");
        return;
    }
    app.PlaySound(Lang::Sounds::OGG_POPUP);
    app.SendTextToAI("[摇一摇启蒙]");  // 触发指令上行
});
```

P30-4G 和 P30-WiFi 双板对称（`mydazy_p30_wifi_board.cc` 同样位置）。

**触发条件**：
- 仅 `kDeviceStateIdle` 状态接管（其他状态摇一摇被忽略）
- 加速度计冷却 1.5s（防误触连发）
- 离线时 `SendTextToAI` 自动唤醒+连接+发文本

### 11.3 摇一摇语义矩阵（按设备状态）

| 当前状态 | 摇一摇行为 |
|---|---|
| **Idle 时钟** | ✅ 触发云端启蒙场景（show_card + LLM 朗读）|
| 故事中（speaking） | 暂忽略（后续可加"切下一段"语义）|
| 教育卡显示中 | 暂忽略（后续可加"切下一张"语义）|
| Listening | 忽略（避免干扰 ASR）|
| Player 模式 | 暂忽略（后续可加"切歌"语义）|
| WifiConfig / Activating | 忽略 |

**当前 P0 仅 Idle 触发** —— 主流程跑通后再扩展其他状态语义。

### 11.4 云端 LLM Prompt 设计（待服务端实现）

```
当用户消息以 [摇一摇启蒙] 开头时（设备 shake 触发）：

【步骤 1】查用户档案
  - age: 3-9 岁年龄段
  - learned_words: 已学英文词 + 时间戳
  - learned_hanzi: 已学汉字 + 时间戳
  - last_review_ts: 上次复习时间

【步骤 2】艾宾浩斯曲线判定（哪些词该复习）
  - 学过 1 天内：权重 100（高）
  - 学过 3 天：权重 70（中）
  - 学过 7 天：权重 40（低）
  - 学过 30 天：权重 20（更低）

【步骤 3】策略选择
  - 70% 概率：推荐新场景
    * 按年龄选 type：3-5 岁优先 hanzi/pinyin，6-9 岁加 word/math
    * 按主题轮换：动物→水果→颜色→家庭→自然→交通...
  - 30% 概率：从该复习的词中选一个

【步骤 4】调用 mcp.tools.show_card
  - 设备端弹卡显示

【步骤 5】TTS 朗读（≤3 句）
  - 新场景：朗读 main + 简单情境句
    "鸟，niǎo，小鸟在天上飞，你看到过几只小鸟？"
  - 复习场景：用学过的词造句强化
    "还记得 apple 吗？小明吃了一个 apple，apple 是红色的。"
```

### 11.5 加深印象（云端艾宾浩斯）
