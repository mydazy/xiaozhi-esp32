3 共享 LDO 关键拓扑（GPIO9）

P30-4G 硬件设计选择：codec + ML307R + LCD 共用一个 AUD_VDD-3.3V LDO · GPIO9 拉低 = 三者同时掉电 · 拉高 = 三者同时通电 · 这是整块板"硬复位"的唯一路径。

触发"硬复位"的路径

场景触发点动作
OTA 升级esp_register_shutdown_handler(ShutdownHandler)esp_restart 前 PA 静音 20 ms → GPIO9=0 → 500 ms 等 22µF 电容放电 → 重启后三者一并冷启
恢复出厂 / 9 连击确认同上（统一钩子）同上
网络切换 SwitchNetworkType同上 + esp_restart()同上 · 让 ML307R 也彻底重启避免 modem 状态残留
深度睡眠 EnterDeepSleepShutdownTouchAndAudioForSleepGPIO9=0 + rtc_gpio_hold_en · 穿越 sleep 维持 LOW
开机InitializeGpio先 hold_dis 再 set_level(0) 锁 10 ms → set_level(1) → 200 ms 等 ES8311/ES7210 稳定
风险点：GPIO9 拉低瞬间会"踢"4G PPP 链路 · 所以 EnterDeepSleep 必须先 ResetProtocol（500 ms） + AT+CFUN=4（800 ms） 才能断电 · 顺序错误会导致服务端 session 残留 + modem 假死。

4 构造函数 11 步时序

这是整块板的"冷启动主入口"· 顺序敏感 · 改动前先去 p30-architecture.md § 一.5 重审。

关键顺序约束

  • InitializeGpio 必须在 HandleWakeupCause 前：HandleWakeupCause 内部要读 BOOT 电平（死按防护），背光 / LDO 也要在此完成软启动。
  • PrepareTouchHardware 必须在 InitializeSpi 后吗？不是：触摸走 I2C，但 attach_lvgl 必须在 Display 已创建后 · 所以触摸驱动 new 在第 5 步，attach 在第 8 步，分两步是刻意的。
  • PowerSaveTimer 在 InitializeButtons 前：按键 lambda 内会调 WakeUp() → power_save_timer_->WakeUp()，反过来会 null deref。
  • GetAudioCodec() 显式触发懒构造：紧接着 ApplyDefaultSettings 要写 audio/output_volume，codec 必须已存在；不在构造函数末尾静默放弃会留隐式依赖。
  • StartWelcomeTask 必须最后：依赖 audio_codec_ + display_ + LVGL 首帧都就绪。

5 HandleWakeupCause · 4 分支

各分支详解

分支first_boot_核心逻辑体感
EXT0 (BOOT)true（通过 1.5 s）① RTC 时戳 + GPIO0 LOW < 500 ms → 短路 sleep · ② CheckBootHoldOnWakeup(1500 ms) 50 ms 步进轮询 · ③ 中途松开 → EnterDeepSleep(true)，不返回主动按 1.5 s 才开机 · 死按多久都安全
EXT1 (SC7A20H)trueesp_sleep_get_ext1_wakeup_status() 仅打日志，不做校验 → 直接走 Init拿起设备 → 立即用 · NVS pickupWake 控开关
TIMERtrueAlarmManager::MarkTimerWakeup() → 让闹钟模块知道当前是定时唤醒，可以用未校时 RTC fallback 触发闹钟到点 · 与 EXT0/EXT1 并存（深睡时三源 arm）
default仅 POWERON / USB / SW 为 truepanic / wdt / brownout / TASK_WDT 等异常重启不放欢迎音，避免"崩溃后假装一切正常"插 USB 自动开机 · OTA 重启静默回 Idle · 异常重启无声
死按防护设计意图：用户深睡时握着设备不放 → BOOT 始终 LOW → EXT0 立即触发开机 → CheckBootHoldOnWakeup 等不到松手 1.5 s 又回深睡 → 唤醒/睡眠死循环。 500 ms 短路 sleep 让这个循环以 sleep 状态稳定下来（< 1 mA），用户松手再按才真开机。

6.2 grace 开机长按补丁

boot_hold_grace_active_ 状态机：

  • 入口（InitializeButtons 顶部）：first_boot_ && BOOT==LOW → 主动 busy wait 最多 5 s 等用户松手 · 至少 store(true) 标志已置 · 用户在此期间不会触发 3 s 关机。
  • 出口：首次 OnPressUp 触发时 exchange(false) · 再按再松不影响。
  • 原因：开机长按 1.5 s 后 InitializeButtons 才注册回调 · 用户可能仍未松手 · 不补丁会立即被 OnLongPress(3000) 当作关机意向（差 1.5 s 而已）。

6.3 长按 3 s + 800 ms 缓冲设计

时间点动作取消机制
0 ~ 3000 ms无响应松开 = 无事发生
3000 msAlert + OGG_REBOOT + esp_timer_start_once(800 ms)松开 → OnPressUp 清 shutdown_armed_ + stop timer
3000 ~ 3800 ms等 timer 触发同上
3800 mstimer 回调 → Application::Schedule 派发 ShutdownOrSleep · 1500 ms 后真关无法取消
为何 timer 不直接关机：ESP_TIMER_TASK 高优先级共享任务 · ShutdownOrSleep 内部含 1.5 s vTaskDelay + AbortSpeaking · 阻塞 timer 任务会卡住整个 esp_timer 调度。所以 timer 回调只做 Schedule 派发 → main_app 任务执行真关机。

9 ShutdownHandler · esp_restart 钩子

通过 esp_register_shutdown_handler 在构造函数开头注册 · ESP-IDF 框架在所有 esp_restart() 调用前自动执行 · 不再为每个重启路径手动写"关 LDO"代码。

三步流程

步动作耗时目的
1gpio_set_level(AUDIO_CODEC_PA_PIN, 0)—静音功放 · 避免下一步断 LDO 时音圈瞬态电压释放产生 POP 异响
2esp_rom_delay_us(20*1000)20 ms等功放进入静音状态（shutdown 上下文用 ROM delay 比 vTaskDelay 更稳）
3gpio_set_level(AUDIO_PWR_EN_GPIO, 0) + rtc_gpio_hold_en—断 LDO 总开关 · hold 让 LOW 穿越 esp_restart() 维持
4esp_rom_delay_us(500*1000)500 ms等 22 µF 电容彻底放电 · 让 codec / 4G / LCD 真正断电复位

覆盖的重启路径

  • OTA 升级（Ota::Reboot）
  • 恢复出厂（SystemReset::CheckButtons(true) + 3 s 倒计时 → esp_restart）
  • 网络切换（SwitchNetworkType 末尾）
  • panic / wdt / brownout（IDF 框架自动重启路径）
统一设计的价值：任何 esp_restart 都会让 ML307R 跟着冷启 · 不会出现"软重启 ESP 但 modem 卡在旧 PPP 状态"的诡异故障。
