这台设备是面向儿童的 AI 语音硬件。其交互核心是一台 设备状态机（device_state.h 共 11 个枚举态，本板实际驱动其中 5 个）， 被四类物理输入源（按键手势 / 触摸手势 / 摇一摇 / 拿起）驱动状态流转；之上叠加一层闹钟响铃最高优先级， 以及 MP3 播放 / 番茄钟两个活动态。再外层是一套贯穿电源域（共享 LDO）的开机—运行—睡眠—关机生命周期。下图按"输入源 → 状态机 → 电源生命周期"三层展开。

一、功能全景 构造函数初始化顺序 + 能力模块

📶 双网络 DualNetwork

二、核心交互状态机 输入源 → 状态流转（在线对话主回路）

Idle 是中枢。所有唤醒最终落到 ToggleChatState() 推进 Idle→Connecting→Listening； 打断回路把 Speaking/Listening 拉回 Idle。注意提示音被设计成"松手信号"，唤醒经 ScheduleWakeChatToggle() 异步等播完再 Toggle，避免在按键/LVGL 任务里阻塞。

★ 实线=普通流转，橙线=高频热路径。Speaking/Listening 的"打断"是儿童最熟悉的退出心智，双击触摸 = 直接回 Idle 并播 exitchat 音。

四、电源 / 睡眠生命周期 共享 LDO(GPIO9) 贯穿全程 · 本项目头号风险区

充电中 跳过深睡 但保持降亮（防发烫）。ShutdownHandler 是注册到 IDF 的关机钩子：无论关机、出厂复位还是切网 esp_restart() 都会经过它统一断 LDO，确保 LCD/音频/4G 三者硬复位干净。

五、三轮评审结论 正确性 → 并发/边缘 → 电源域红线

第 1 轮 · 流程正确性

OKIdle 作为唯一中枢，所有唤醒收敛到 ToggleChatState

按键 / 触摸 / 摇一摇三条唤醒路径，最终都通过 ScheduleWakeChatToggle() 异步推进，把"等提示音播完"和"状态切换"解耦，避免在 button/LVGL 任务里 vTaskDelay 卡死。设计自洽。

观察OnStrike 已定义但未挂载

桌面双击(敲击)回调 OnStrike 写了"停闹钟 / 唤醒屏幕"逻辑，但 InitializeSc7a20h 只注册了 sc7a20h_shake(...&OnShake...)，没有注册 strike。属预留或被摇一摇取代的死代码——建议确认意图，要么挂载要么删除以免误导。

OKConnecting 打断有兜底

按键打断握手时先 CloseAudioChannel()（靠回调切 Idle），又额外 Schedule 一个兜底：若 channel 未真打开、回调不触发，则强制 SetDeviceState(Idle)，防止永久卡在"连接中"。考虑周到。

第 2 轮 · 并发与边缘场景

OK三处原子防重入

shutdown_armed_.exchange(true) 防长按重复触发关机；boot_hold_grace_active_ 防开机长按被误判成 3s 关机；配网切换用 atomic_flag test_and_set 防双任务。均符合 ~/.claude/rules "多核共享用 std::atomic" 红线。

留意出厂复位 10s 窗口依赖手动清理

waiting_factory_reset_confirm_ 的超时（10s）在 OnClick/OnDoubleClick lambda 里惰性判断清除，没有独立定时器。若 9 连击后用户既不双击也不再点任何键，标志会一直挂着直到下次按键——非 bug，但状态不会自动复位。可接受（下次任意点击即清）。

留意dead-hold 短路窗口仅 500ms

从 sleep 到再次按键 < 500ms 且仍按住 → 判定"用户没松手"直接回深睡，防误开机。窗口偏短但配合开机长按 1.5s 双保险，逻辑成立。RTC 变量 s_last_sleep_us 用 RTC_DATA_ATTR 跨深睡保持，正确。

第 3 轮 · 电源域红线（CLAUDE.md 头号风险）

OK开关机时序严防 LCD 闪烁与喇叭 POP

开机：LDO 先锁低 → 再释放 RTC hold（消除浮动窗口）→ 1s VBAT 放电 → 拉高，避免冷启动毛刺。关机：ShutdownHandler 先 PA 静音 20ms → 断 LDO → RTC hold 穿越 restart → 等 22µF 电容放电。两端都在守电源毛刺红线。

观察共享 LDO 是单点：音频+4G+LCD 同生共死

AUDIO_PWR_EN(GPIO9) 一脚控制三大耗电件，断电即三者硬复位。当前靠它实现"无独立复位脚"的复位，但也意味着任何一路异常只能整组断电。代码已在注释标注 TODO 下版加 ML307 独立电源(VT4 栅极)，与硬件演进对齐。属设计约束而非缺陷。

OK深睡前优雅释放 4G/网络，避免服务端残留 session

EnterDeepSleep 与 SwitchNetworkType 对称：都先 ResetProtocol() 优雅断 MQTT/WS，再 AT+CFUN=4 释放 PPP/RRC（等 800ms），最后才断音频电源。顺序正确——必须在 I²C/音频断电之前 arm 陀螺仪 EXT1，代码注释明确标了这条依赖。
