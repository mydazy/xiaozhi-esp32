3 跨线程数据流

教育卡 / 笔画 GIF 数据来自 2 个 FreeRTOS 任务，全部通过 Application::Schedule() 投回 main loop。

关键不变量

  • active_scene_ / in_font_mode_ 只在持 DisplayLockGuard 的 main loop 内更新
  • LVGL 渲染调用必须持 DisplayLockGuard（lv_timer / 异步事件除外）
  • FontGif 内 PSRAM buffer 所有权转移给 LvglRawImage（owns_data=true · 析构 heap_caps_free）
  • OnFontExitClicked / OnEduCardClicked 用 Schedule() 异步退出，避免 LVGL 事件回调内改 src 触发链问题
  • 无 g_edu_queue / 无 esp_timer：单卡架构 = 无队列状态 · 无 timer 生命周期 = 零跨轮泄漏

14 环节评审：14/14 全部 ✅ · 旧 4 处 ⚠ Hack 全部消除（c-1 / c-2 / d-1 / e-1）。
🔴 视觉遮挡 bug 闭合（2026-05-12 · 实测优先）
用户实测现象：调用一次 FontGif 后，之后所有对话底部都不显示字幕。 根因（不是 token race）：
  1. FontGif() 内 lv_obj_move_foreground(emoji_box_) 把 emoji_box 提到屏幕最前 z-order
  2. HideFontGif() → ShowBottomBar() 旧版仅 lv_obj_remove_flag(bottom_bar_, LV_OBJ_FLAG_HIDDEN)
  3. bottom_bar flag=visible · 但物理 z-order < emoji_box → 视觉被永久遮挡
  4. 后续 SetChatMessage 父类 remove HIDDEN 也未 move_foreground → 字幕永远看不见
修复：ShowBottomBar() 内追加 lv_obj_move_foreground(bottom_bar_)。 对照：SwitchToChatMode（line 391）和 HideEduCard（line 1015）早已用同款 move_foreground 思路，ShowBottomBar 是漏的一环。
🔴 跨轮 race 闭合（2026-05-12）
缺陷：HideFontGif() 只清 in_font_mode_ + font_pending_，未推进 font_request_active_。 触发时序：
  1. T0：BeginFontPending → active=N · MCP show_stroke 异步下载启动
  2. T1：用户唤醒 → kDeviceStateSpeaking → HideFontGif()（active 仍=N）
  3. T2：下载迟到完成 → Schedule FontGif(buf, size, N)
  4. T3：FontGif 内 token 检查 active=N → 匹配 → 装载 + HideBottomBar() + in_font_mode_=true
  5. T4：后续 LLM 字幕 SetChatMessage("assistant", ...) 被守护吞 → 字幕永久看不见
修复：HideFontGif() 增加 font_request_active_.store(0, release)。后续 FontGif(N) 进入时 active=0, N≠0 → stale → heap_caps_free(buffer) + return。 幂等性：BeginFontPending 用 store(id) 直接覆盖，不依赖旧值；新一轮 show_stroke 流程不受影响。
✅ 总评：6/6 路径功能完整 + 视觉流畅 + 跨线程零 race + 清场动作清晰 + 零 hack。

量产纪律

  • DeviceState 11 态不新增枚举 —— 22 项功能盘点 0 项需要新增
  • SceneType 8 值不轻易扩展 —— 真有第 6 个共享逻辑全屏 Scene 时才抽 SceneFactory
  • 不抽 ActivitySession / SceneNavigator 基类 —— LVGL screen 已是 Scene 抽象
  • 事件型扩展走 AudioSource enum —— Alarm / PomodoroBell / Reminder 禁加 IsXxx flag
  • YAGNI 原则 —— 不为未来扩展提前留基础设施（已删 3500ms 链式 timer 即此教训）
  • 新模块在 docs/p30-architecture.html § 一.5.2 落位表登记（A×B×C 三维坐标）
