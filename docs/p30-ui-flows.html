3 跨线程数据流

教育卡 / 笔画 GIF 数据来自 2 个 FreeRTOS 任务，全部通过 Application::Schedule() 投回 main loop。

关键不变量

  • active_scene_ / in_font_mode_ 只在持 DisplayLockGuard 的 main loop 内更新
  • LVGL 渲染调用必须持 DisplayLockGuard（lv_timer / 异步事件除外）
  • FontGif 内 PSRAM buffer 所有权转移给 LvglRawImage（owns_data=true · 析构 heap_caps_free）
  • OnFontExitClicked / OnEduCardClicked 用 Schedule() 异步退出，避免 LVGL 事件回调内改 src 触发链问题
  • 无 g_edu_queue / 无 esp_timer：单卡架构 = 无队列状态 · 无 timer 生命周期 = 零跨轮泄漏

P0 问题统一根因：FontGIF 借用 emoji_box_（kChat 的 L0 容器）作为 GIF 渲染载体。在 kChat 场景 emoji_box 本就显示，无 bug；但 kClock/kPlayer/kPomodoro 场景下 SwitchTo*Mode 显式 add_flag(HIDDEN) 隐藏了 emoji_box，FontGIF 处理不对称：

  • 退出 bug：HideFontGif() 没收回 emoji_box · 不重新前台 L0 容器 → emoji_box 滞留覆盖底层
→ 3 个场景全部受影响：kClock / kPlayer / kPomodoro。kChat 例外（emoji_box 本就是主体）。

稳定优先：不动 FontGIF 架构（不改"借用 emoji_box"），只在进入/退出对 emoji_box 做对称处理。

进入修复：FontGif() 内显式解 HIDDEN

if (emoji_box_) {
    // 🔴 修复 P0 进入 bug：番茄钟/Clock/Player 主屏 emoji_box 被 HIDDEN，必须显式解开
    lv_obj_remove_flag(emoji_box_, LV_OBJ_FLAG_HIDDEN);

退出修复：HideFontGif() 末尾场景路由

void UiDisplay::HideFontGif() {
    font_pending_.store(false);
    font_request_active_.store(0);
    DisplayLockGuard lock(this);
    in_font_mode_ = false;
    ShowBottomBar();
    LcdDisplay::SetEmotion("neutral");

    // 🔴 修复 P0 退出 bug：按 active_scene_ 收回 emoji_box · 恢复 L0 容器前台
    auto restore = [&](lv_obj_t* container) {
        if (emoji_box_) lv_obj_add_flag(emoji_box_, LV_OBJ_FLAG_HIDDEN);
        if (container) lv_obj_move_foreground(container);
    };
    switch (active_scene_) {
      case SceneType::kClock:    restore(clock_container_);    break;
      case SceneType::kPlayer:   restore(player_container_);   break;
      case SceneType::kPomodoro: restore(pomodoro_container_); break;
      case SceneType::kChat:    break;   // emoji_box 本就是主体，不动
      default: break;
    }
}
对称性原则：进入和退出对 emoji_box 的处理必须对称——进入解 HIDDEN，退出加 HIDDEN；进入提顶，退出降底。否则会出现"进得去退不出"或"退得出进不去"的不对称 bug。

tick_cb 改造（每秒触发）

SetTickCallback([](State state, uint32_t remain, uint32_t total) {
    bool running = (state == State::kRunning);
    bool active  = (state != State::kIdle);
    Application::GetInstance().Schedule([remain, running, active]() {
        auto& app = Application::GetInstance();
        auto* ui = dynamic_cast<UiDisplay*>(Board::GetInstance().GetDisplay());
        if (!ui) return;

        if (!active) {
            // Stop / Finish → 切回 clock（应用层调 SwitchOutPomodoroMode）
            ui->SwitchOutPomodoroMode();
            return;
        }

        // 🔴 关键：仅在 Idle 状态下切 UI · 对话中只更新已在 Pomodoro 的内容
        if (app.GetDeviceState() == kDeviceStateIdle) {
            if (!ui->IsPomodoroMode()) ui->SwitchToPomodoroMode(remain, running);
            else                       ui->UpdatePomodoro(remain, running);
        } else {
            // 对话 / 配网中：UI 让位 · 番茄钟后台 tick · UI 不切走 Chat
            if (ui->IsPomodoroMode()) ui->UpdatePomodoro(remain, running);
        }
    });
});

🔴 时钟主屏卡死 bug 闭合（2026-05-12 · 实测最高优先级）
用户实测现象：调用一次对话后回到时钟主屏，下次按键/唤醒不切换到表情，UI 卡在时钟页面。 根因（不是 UI 层 bug · 是状态机调度跳过）：
  1. 对话结束后 protocol_ 的 audio channel 不关闭（复用以减少建连延迟）
  2. 第二次唤醒 HandleToggleChat / HandleStartListening / HandleWakeWord 内 state==Idle && IsAudioChannelOpened() 分支 直接 SetListeningMode → SetDeviceState(Listening)， 跳过 SetDeviceState(Connecting)
  3. 主循环消费 STATE_CHANGED 时 GetState()=Listening → 直接走 case kDeviceStateListening， 跳过 case kDeviceStateConnecting 内的 SwitchToChatMode()
  4. case kDeviceStateListening 旧设计"不清场"（保留 EduCard/FontGif），不调任何 UI 切换
  5. active_scene_ 保持 kClock · clock_container_ 全屏 LV_OPA_COVER 不透明覆盖 · emoji_box_ 被隐藏 → 用户看不到表情
设计意图（application.cc:886 注释）：// Schedule to let the state change be processed first (UI update) —— channel 未开路径已遵循此设计（Connecting + Schedule 异步开 channel），但 channel 已开路径漏掉了"先走 Connecting"。 修复：case kDeviceStateListening 内追加 if (lcd) lcd->SwitchToChatMode() 兜底。 SwitchToChatMode 早退守护 if (active_scene_ != kClock) return 保证幂等：
  • 正常 Connecting→Listening：active_scene_=kChat → 早退（无副作用 · 保留 EduCard/FontGif）
  • Speaking→Listening 同轮继续问：早退（符合"Listening 不清场"设计）
  • Idle 直跳 Listening（channel 已开）：active_scene_=kClock → 切 Chat ✅
影响 3 处入口：HandleToggleChat:892 · HandleStartListening:944 · HandleWakeWord:1001（line 1001 因 ContinueWakeWordInvoke 内 != Connecting return 实际是 dead code，但 case Listening 兜底覆盖）。
🔴 视觉遮挡 bug 闭合（2026-05-12 · 实测优先）
用户实测现象：调用一次 FontGif 后，之后所有对话底部都不显示字幕。 根因（不是 token race）：
  1. FontGif() 内 lv_obj_move_foreground(emoji_box_) 把 emoji_box 提到屏幕最前 z-order
  2. HideFontGif() → ShowBottomBar() 旧版仅 lv_obj_remove_flag(bottom_bar_, LV_OBJ_FLAG_HIDDEN)
  3. bottom_bar flag=visible · 但物理 z-order < emoji_box → 视觉被永久遮挡
  4. 后续 SetChatMessage 父类 remove HIDDEN 也未 move_foreground → 字幕永远看不见
修复：ShowBottomBar() 内追加 lv_obj_move_foreground(bottom_bar_)。 对照：SwitchToChatMode（line 391）和 HideEduCard（line 1015）早已用同款 move_foreground 思路，ShowBottomBar 是漏的一环。
🔴 跨轮 race 闭合（2026-05-12）
缺陷：HideFontGif() 只清 in_font_mode_ + font_pending_，未推进 font_request_active_。 触发时序：
  1. T0：BeginFontPending → active=N · MCP show_stroke 异步下载启动
  2. T1：用户唤醒 → kDeviceStateSpeaking → HideFontGif()（active 仍=N）
  3. T2：下载迟到完成 → Schedule FontGif(buf, size, N)
  4. T3：FontGif 内 token 检查 active=N → 匹配 → 装载 + HideBottomBar() + in_font_mode_=true
  5. T4：后续 LLM 字幕 SetChatMessage("assistant", ...) 被守护吞 → 字幕永久看不见
修复：HideFontGif() 增加 font_request_active_.store(0, release)。后续 FontGif(N) 进入时 active=0, N≠0 → stale → heap_caps_free(buffer) + return。 幂等性：BeginFontPending 用 store(id) 直接覆盖，不依赖旧值；新一轮 show_stroke 流程不受影响。
✅ 总评：6/6 路径功能完整 + 视觉流畅 + 跨线程零 race + 清场动作清晰 + 零 hack。

量产纪律

  • DeviceState 11 态不新增枚举 —— 22 项功能盘点 0 项需要新增
  • SceneType 8 值不轻易扩展 —— 真有第 6 个共享逻辑全屏 Scene 时才抽 SceneFactory
  • 不抽 ActivitySession / SceneNavigator 基类 —— LVGL screen 已是 Scene 抽象
  • 事件型扩展走 AudioSource enum —— Alarm / PomodoroBell / Reminder 禁加 IsXxx flag
  • YAGNI 原则 —— 不为未来扩展提前留基础设施（已删 3500ms 链式 timer 即此教训）
  • 新模块在 docs/p30-architecture.html § 一.5.2 落位表登记（A×B×C 三维坐标）

命名规范（落地汇总）

类命名理由
场景枚举SceneType::kChat（非 kEmoji）产品语义对齐 SwitchToChatMode
笔画入口FontGif()（非 EnterFontMode / UpdateFontGif）名词命名，与 ShowEduCard / ShowQrCode 风格统一
笔画退出HideFontGif() 公开（与 FontGif 对称）application.cc Speaking 状态需显式调用 · 命名对称 Show/Hide 风格
底部条显隐ShowBottomBar() / HideBottomBar() 对称替代 SetBottomBarVisible(bool) · 命名对齐 Show/Hide 族
教育卡分类const char* category（无 EduCategory enum）调用入口都是 LLM 字符串，9 enum 是死代码
渲染主体RenderEduCardLayout（非 BuildEduCard）强调"已知数据→画屏"，避免与 ShowEduCard 混淆
教育卡提取ExtractFirstEduWordCard(optional)（非 ExtractEduWordCards(vector)）单卡架构 · optional 比空 vector 更准确
教育卡投递TriggerEduCard(EduWordHit)（非 EnqueueEduCards(vector)）单卡架构 · 无 queue · 名词直接
状态字段active_scene_ / in_font_mode_消除 "current_xx" 命名重复
设计原则：量产稳定期，实际使用驱动 API 形态，不为对称性或编译期类型安全提前抽象。
