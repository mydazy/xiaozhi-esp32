1 概览

AXS5106L IIC 电容触摸控制器 · 0 内部任务 · 走 LVGL 30ms poll + IRAM ISR 边沿计数 · I2C 经 i2c_bus_worker 串行化（共线 codec/sensor/NFC）。

3 启动 + 复位时序

GPIO 引脚：TOUCH_RST=GPIO_4 · TOUCH_INT=GPIO_5（三板 config.h 一致）。INT 配负边沿 + 内部 pull-up ~45kΩ。

4 手势识别状态机

关键边界

  • jitter_unstable：累计抖动 > 32 屏蔽 tap（防 RF 干扰假帧）
  • last_click_time 清零：manhattan ≥ SWIPE_THRESHOLD 时清，防"tap→拒绝 swipe→tap"被错配为 double（v3 已修）
  • release_count：无触摸帧 ≥ 2 才报 RELEASED（防 4G RF 引起的瞬时丢帧）
  • POST_RELEASE_GUARD：release 后 100ms 内若再 press，作为 double 候选；超时则新事件起点
  • swipe 路径 #if 0 封死：识别代码已删 · 但 enum 值 SWIPE_UP/DOWN/LEFT/RIGHT 还在 .h 公开 API

5.2 STORM 防护（v3.0+）

RF 干扰风暴：4G TX 期间 INT 引脚被脉冲打穿可能产生 > 12 edges/s 假中断。检测到 storm 时 mute 2 秒（不读 I2C 不投递回调）· INT 一直 disable · 等 baseline 重置。

5.3 唤醒回调时机

场景wake_cb 是否触发
新触摸 press-edge✓
同一长按持续中❌（仅边沿）
POST_RELEASE_GUARD 期间触摸下一帧才触发（30ms 延迟）
storm_detected 期间❌（被 mute）

6 深睡进出

6.1 进入深睡

步动作
1ShutdownTouchAndAudioForSleep 调 axs5106l_touch_sleep
2sleeping=true（LVGL read 立即跳出）
3gpio_intr_disable(INT)
4I2C 写 REG_SLEEP(0x19)=0x03
5RST 拉高保持
6touch_driver_ = nullptr · 句柄不可再用

6.2 唤醒（非深睡场景 · 暂未主链路使用）

axs5106l_touch_resume：reset_chip + 重置 storm baseline + gpio_intr_enable。当前深睡唤醒走完整重启路径（PrepareTouchHardware 重新调 axs5106l_touch_new），不使用 resume。
