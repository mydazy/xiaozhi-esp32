## 三、交互流程步骤（8 条主流程·代码真实行为）

### 1. 首次开箱（WiFi 版）
上电 → 长按 1.5s 开机 → Starting（开机音）→ 无 WiFi 记录 → 配网模式：
屏显二维码 + 播提示音 → 微信扫码 BluFi 配网（双击屏幕可切热点模式）→
密码验证（连接测试 10s）→ 成功 → **强制重启**（释放 BT 内存）→ 激活（屏显激活码）→ 待机时钟页。
配错密码：BLE 回错误码可重试；10 分钟无操作 → 深睡。
4G 版：插卡免配网，开机 8s 等模组 → 检测注册（最长 ~98s）→ 直接激活。
失败（无 SIM/欠费）：Alert 提示 → 转待机（本轮修复，原永卡 Starting）。

### 2. 日常唤醒对话
待机时钟页 → 喊唤醒词 → 唤醒音(wakeup.ogg) + 切聊天页 → Connecting（建 wss ~1-3s）→
Listening（录音上行）→ 服务端 ASR+LLM → Speaking（TTS 下行 + 字幕 + 表情）→
说完回 Listening（实时模式可直接追问）→ 静默超时回 Idle 时钟页。
对话中喊唤醒词 = 打断 TTS，播 wakeup 音重新听（本轮修复了打断反馈音）。

### 3. 教育卡片/笔画
对话中 AI 回答携带卡片标记 → 字幕区上方弹出卡片 overlay（标题+内容两行）；
笔画指令 → 后台任务下载 GIF（HTTPS·PSRAM）→ 全屏播放笔画动画 → 场景切换退出。

### 4. 闹钟
语音设闹钟 → MCP 校验（消息 ≤10 汉字否则报错话术）→ 写 NVS →
空闲自动深睡时：闹钟 >30min → RTC 定时唤醒（≤4h 提前 30min 醒等待；>4h 分段每 2h 醒校时）；
闹钟 ≤30min → **放弃深睡保持运行**（本轮修复，原睡死漏响）→
到点响铃（每 5s 一次）→ 任何按键/触摸/摇晃 = 停铃（优先级高于对话）。

### 5. 休眠与唤醒
Idle 60s 无操作 → 降背光；300s → 深睡流程（停音频→断协议→4G 飞行模式→arm 唤醒源→深睡）。
唤醒源：BOOT 键（EXT0·常开）+ 闹钟 RTC + 拿起（EXT1·默认关）。
深睡后唤醒 = 全新开机（~10s 到待机）。充电中不深睡。
> 2026-06-11 决策：**维持深睡+按键唤醒，不做"息屏断网+保留唤醒词"语音待机**。
> 推演依据：唤醒词常听需主芯片不睡，整机 ~75-125mA，1000mAh 电池纯待机仅 ~8-13h（深睡为周级）。
> 硬约束：GPIO9(AUDPWR-EN) 是 ES7210麦克风+ES8311+LCD+ML307R 共用电源总闸，保唤醒词必须整轨供电。

### 6. 充电与低电
1Hz 电池采样（充电/放电双查表）→ 低电 ≤5% Alert + 提示音 →
3.4V 连续 3 次确认 + 非充电 → 强制关机（保护电芯）。
已知缺口：未校准机 Vref 偏高 16% 该关不关（08-P0-A 方案待批）；"关机"后 4G 模组仍 ~5mA 暗耗（硬件限制）。

### 7. OTA 升级
仅开机激活阶段检查版本（对话中不触发）→ 有新版 → 屏显升级中（禁对话）→
下载（断点续传·弱网指数退避）→ 校验写入 → 重启 → 新固件启动即标记有效（本轮修复，原弱网会被回滚）。

### 8. 控制中心（2026-06-11 改版 · 两板布局/习惯完全统一）
待机/对话页点状态栏 → 控制中心 3×2 宫格：
第一行【网络 · 打断(AEC) · 休眠】，第二行【退出 · 亮度 · 关于】。音量不进控制中心（物理按键+语音调节）。
- **网络按钮（二次确认防误触）**：第一次点变橙色"再点确认"，3s 不点自动还原；二次点确认后执行——
  4G 版=切 WiFi/4G（写 NVS+重启），WiFi 版=进/退配网（同 BOOT 3 连击语义，重启进配网）。
- **打断(AEC) 开关**：触屏与 BOOT 双击走同一底层 ToggleAecMode（本轮修复 4G 板触屏断链：
  原切换逻辑内联在按键处理里未接虚函数，控制中心调到基类空函数）。仅 Idle/Listening/Speaking
  态可切（激活/配网/升级中点击无效，属保护）；UI 不预翻状态，以切换后真实值回写（防显示撒谎）。
- **工程纪律**：所有重操作（切网/休眠开关 NVS/AEC）经 Schedule 调度到主线程，LVGL 线程零阻塞；
  默认亮度 35% 与系统出厂值一致；ManagedTimer CreateOnce 补 auto_delete=false（LVGL9 默认回调后
  自动释放 timer，原实现存在 double-free 地雷）。

### 🟡 困惑/抱怨级（精选）
- 开机长按 1.5s 零反馈（按住不亮灯）→ E4"长按5-6秒、开关机像反了"；首启 1kHz 校准蜂鸣无解释
- 配网"双击切换模式"指按键，家长会去戳屏幕（没反应→以为触摸坏了）；密码错只回英文 "Connection failed"
- 配网 10 分钟超时→3 秒提示→黑屏深睡（"新机自己关机了"）
- 激活期每 1-2 分钟以 80% 音量循环朗读激活码，无法退出
- 第一次使用零引导：屏幕只有时钟，没人告知唤醒词是"搭子精灵"
- 出厂音量 80% 对儿童设备偏吵（C2/G1 同向）；无家长音量上限/使用时长管控
- 闹钟一律 80→100% 渐强必醒模式，温和提醒模式（kReminder）代码完整但零调用；响铃时戳屏幕不停铃反而拉起对话
- 番茄钟完成只响一声，错过就错过；误按一下按键无确认清零 25 分钟进度
- 4G 版从第一天起就看不到笔画动画（4G 直接跳过 GIF 下载）
- "关机"后 4G 模组暗耗 5mA，放一周必没电（"放着也亏电"→归因电池差）；低电 5% 才提醒≈无预警
- OTA 无电量门槛（低电升级中途没电白下）；升级后 MIC 重校"哔"一声无解释；报错把 code/url 糊用户脸上

### 🟢 验证合理的设计
夜间深睡静音（pickupWake 关闭后桌面震动不再吵人）、按键误触防护（短按静默回睡）、
追问免唤醒词、闹钟优先级高于对话（任何输入先停铃）、断网自动进配网兜底、
OTA 失败恢复旧版+防误回滚（本轮修复）、教育卡片防呆链路。

1. **产线核对**（零代码）：4G SKU 是否预写 NVS `network/type=1`——不确认则卖点开箱即断。
