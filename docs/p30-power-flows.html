3.3 EnterDeepSleep 11 步（v2.2.11 修复顺序）

总耗时 ~3.6s（4G）/ ~3.0s（WiFi）· 橙色为板差异点 · 绿色为关键顺序点。

⚠ 量产实测 bug（2026-05-12 修复，v2.2.11）：原代码把 sc7a20h_arm_wakeup 放在 ResetAllGpiosForSleep 之后 · I2C 引脚（GPIO11/12）已被 reset · 驱动里 i2c_worker_write_read(INT1_SRC) 静默失败（早期版本 (void) 吞了返回值， v2.2.11 加了 LOGW 诊断）· INT1 锁存位（CTRL_REG5=0x08 LIR_INT1）残留 LOW · esp_deep_sleep_start 一看 GPIO3 已经 LOW → ext1 立即触发 → 设备瞬间重启。

修复要点：
  • SC7A20H 与 audio codec 共用 I2C_NUM_1（SDA=11 / SCL=12）· 顺序必须严格
  • 音频电源切断（AUDIO_PWR_EN=0）→ 留 ≥200ms 让机械振动衰减后再读 INT1_SRC
  • arm_wakeup 紧贴 esp_deep_sleep_start 之前 · 中间不能再做任何 vTaskDelay/log
  • EXT1 仅由 arm_wakeup 负责 · ConfigureDeepSleepWakeupSources 只管 EXT0（避免重复 arm）
P31 板已知缺陷（待 P31 切换 v4.0 C 驱动后修复）：P31 用旧 C++ Sc7a20h 类 inline arm EXT1，从未读 INT1_SRC 清 latch。已在 main/boards/mydazy-p31/mydazy_p31_board.cc EnterDeepSleep 加注释。

4 自动休眠 · PowerSaveTimer

三板配置统一：PowerSaveTimer(120, 60, 300) · 三参语义：(idle_to_save_s, debounce_s, save_to_shutdown_s)

设计意图 · 配网态 + 未激活态也走自动休眠（不是 bug）：防"用户忘记操作 → 设备整夜耗电亏电变砖"。用户拿起或按 BOOT 键即恢复。

5 唤醒源

源触发校验体感
EXT0BOOT 键低电平必须长按 ≥1500 ms · 否则立即 EnterDeepSleep主动按键开机
EXT1SC7A20H INT1（拿起 / 移动）无校验 · 直接 Init拿起即用 · NVS pickupWake 控制
TIMERRTC 定时设 is_alarm_clock=true（未使用）v33 闹钟功能后启用
defaultRST / USB 插入 / Watchdog—插 USB 会自动开机（充电友好）

ConfigureDeepSleepWakeupSources（v2.2.11 后只负责 EXT0）

  1. arm EXT0：esp_sleep_enable_ext0_wakeup(BOOT_GPIO, 0) + 上拉
  2. delay 200ms 让 RTC GPIO 配置生效

用户死按 BOOT 不松手 → 立即被 EXT0 触发重启 → CheckBootHoldOnWakeup(1.5s) → 显示 logo + 欢迎音 → 用户自然松手 · 链路自愈 · 不需要等松手兜底。

sc7a20h_arm_wakeup（v2.2.11 后唯一的 EXT1 入口）

  1. 读 REG_INT1_SRC（清 latch · 失败时 LOGW 出诊断）
  2. esp_sleep_enable_ext1_wakeup_io(SC7A20H_GPIO_INT1, ANY_LOW)
  3. rtc_gpio_pullup_en(INT1) + rtc_gpio_pulldown_dis(INT1)

必须紧贴 esp_deep_sleep_start 之前调用 · I2C 总线必须仍存活（GPIO11/12 不能先被 reset）。
