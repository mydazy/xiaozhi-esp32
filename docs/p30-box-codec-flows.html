2 硬件拓扑 · ES8311 + ES7210 + PA + I2C 共线

两个 codec 共用 I2S0 + I2C bus_worker · ES8311 走 STD（DAC mono） · ES7210 走 TDM 4-slot（MIC×2 + REF + 空） · PA pin 由 ES8311 driver 内部控制。

关键观察：① ES8311 / ES7210 两个 codec 走同一个 i2c_bus_worker · 4G RF 共线场景下不会产生协议层污染（mydazy_codec_ctrl_i2c.c 设计意图）。 ② PA pin 由 ES8311 driver close 时自动拉低 · POP 噪声防御依赖 EnableOutput 的 mute + 30ms 延迟。 ③ MIC1 是唯一显式设 gain 的通道，MIC2 / REF 走 driver 默认值（待验证 V-2）。

7 MIC 校准决策树（in_raw → input_gain / mic_type / aec_gain）

第一轮 RMS 推导一切 · 输出 3 个参数（input_gain dB · mic_type dBV · aec_gain dB） · 三档式映射保证物料 -26/-36/-42 dBV 全覆盖。

1758 的推导（C-4 缺注释项）：mic_type=36 dBV、input_gain=15 dB 时校准后等响度 RMS=5000。设当前 input_gain=15 dB 实测 RMS=rms，则 mic_type(dBV) = 36 + 20·log10(1758/rms)，其中 1758 = 5000 / 10^((36-15)/20)。本质：1758 是"中间档基准 RMS"，用反推算出当前物料的实际 dBV 灵敏度。
