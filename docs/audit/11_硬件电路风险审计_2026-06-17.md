# 11 P30 硬件电路全板风险审计（2026-06-17）

> 审计对象：MyDazy-P30 硬件（KiCad 原理图 + PCB + 网表重建）。
> 数据来源：`outputs/p30-rebuild/`（netlist.json 323 元件 / 304 网络、pdf_vals.json 器件值、ERC）。
> 方法：四方法分工 —— ① 网表静态连接性检查（全 304 网络）② 电源域公式核算（datasheet 为准）③ SPICE 仿真（模拟子电路）④ 硬件↔固件交叉审计。
> 纪律：电源域为红线，所有数值均 grounded（网表拓扑 + datasheet + SPICE），凡需上板实测的项明确标注「待实测」，不脑补。

---

## 〇、为什么不是「全板 SPICE」

P30 共 17 颗 IC，绝大多数是数字 / 功能芯片（ESP32-S3 主控 U53、4G 模组 U66、音频 codec U62、触摸 U61、USB 切换 U73/U74、NFC、LCD），**没有 SPICE 模型，行为取决于固件与协议，无法也无意义用 SPICE 仿**。真正可 SPICE 的只有少数模拟网络。因此"全板覆盖"靠四方法分工，SPICE 仅其一。

## 一、电源树架构（grounded，来自网表拓扑）

```
USB-C(J10) ── VBUS_5V ──[D2/D22]── VBUS_5V_P ──► U1 (OVP/负载开关) ──► VIN
                                                                        │
                              VIN ──► U56 (TP4054 线性充电) ──► VBAT_4V2 (电芯)
                              VIN ──► U55 (LDO) ─────────────► VDD_3V3 (主 3V3, 39 脚)
                              VIN ──► U58 (LDO, EN=AUDPWR_EN) ─► AUD_VDD_3V3
                              VIN ◄─► Q15 (电源路径 MOS) ◄─► VBAT_4V2
                              VBAT_4V2 ──[R250 1M]──┬── VBAT_DET ──► ESP32 ADC(U53.13)
                                                    └──[R687 1M]──GND  (+C350 0.01µF)
充电状态： U56.1 (CHRG, 开漏) ──[R570 1K]──► GPIO21 (固件 charging_pin)
```

电源域分多路：`VDD_3V3`(主)、`AUD_VDD_3V3/AUD_3V3/CODEC_3V3`(音频)、`SPI_3V3`、`USIM_VDD`(4G SIM)、`CC_VDD`(Type-C CC)、`VDD_EXT`。结构合理，无明显电源网冲突。

## 二、分级风险清单（核心）

| 编号 | 级别 | 子系统 | 问题 | 依据 |
|---|---|---|---|---|
| **HW-P1-A** | P1 | 电池采样 | VBAT_DET 分压 **1M//1M = 500kΩ 源阻抗**，远超 ESP32-S3 ADC 推荐（≪100kΩ）。SPICE：50nA 漏电即造成电池读数 **−50mV 常数偏移**，且对漏电/温漂敏感 | netlist + SPICE |
| **HW-P2-B** | P2 | 充电 | TP4054 PROG=R571 2.2K → **充电电流 ≈455mA ≈0.45C**（1000mAh 电芯）。线性充电压差大时单封装发热（5V→3.7V×0.45A≈0.6W），密闭机内延长充电/发热 | datasheet 公式 |
| **HW-P2-C** | P2 | 充电检测 | TP4054 **只有 1 个 CHRG 开漏状态脚**（U56.1），硬件层面无法区分"充电中 / 已充满涓流"。这是固件 08-P2-E 的硬件根因——固件改不掉 | netlist + datasheet |
| **HW-P2-D** | P2 | NFC | `WS_IRQ`(NFC1.3) 去噪后是**唯一真·单脚网络**——NFC 中断脚未接到 MCU GPIO。若设计为轮询则可接受；若应中断驱动则为缺陷 | netlist 去噪 |
| **HW-P3-E** | P3 | 主晶振 | 40MHz(Y1) 负载网络 = C727 22pF + **串联电感 L644** + C728（值被脏数据覆盖）。22pF 是否匹配晶振 CL、L644 意图、C728 实际容值待确认 | netlist |
| **交叉确认** | — | 电池采样 | 硬件分压 **÷2** 与固件 `voltage*2` **一致**（08 审计"漏乘×2"假阳性再次证伪）。分压比无误 | netlist↔固件 |

> 说明：本轮硬件侧**未发现会立刻烧硬件/砖机的 HW-P0**。电源头号红线仍在固件侧 **08-P0-A**（见第五节交叉矩阵）。

## 三、四方法各自结论

**① 网表静态检查（全 304 网络）**
- 单脚网络 101 个，**100 个是 U66(4G,109 脚)等多脚模组的 `Net_NNN` 重建噪声**，非真悬空（PCB→原理图重建未给模组内部网命名所致）。去噪后真·单脚命名网络仅 1 个：`WS_IRQ`（→ HW-P2-D）。
- 电源网络脚数合理：GND 203、VDD_3V3 39、VIN 25。无电源多源短接迹象。
- I2C 主总线挂载：U53(MCU)、U62(codec)、NFC1，带上拉(R582-585)与 ESD(D49/D50)；触摸 U61 走**独立** I2C0 总线。设备数少，地址冲突风险低（具体地址待 codec datasheet 核 AD0/AD1）。
- 去耦：134 颗电容，主要电源网密布 100nF + 大容量(10µF/22µF/100µF)，覆盖充分。

**② 电源域公式核算**
- 充电电流：TP4054 `I=1000/R_PROG(kΩ)`，R571=2.2K → **455mA**（→ HW-P2-B）。
- 复位：MCU_RESET = R686 10k 上拉 + C189 1µF + KEY4，RC≈10ms，**标准 ESP32 EN 配置，无问题**。
- 晶振：18pF(C175/C176) 属 **32.768kHz RTC(X3)**，正常；40MHz 用 22pF + L644（→ HW-P3-E）。
  - ⚠️ 订正前期口误：早先"18pF 配 40MHz 偏大"为**误判**——18pF 是 RTC 的，不是主晶振的。

**③ SPICE 仿真（VBAT_DET 子电路，ngspice 实跑）**
- DC：分压输出 4.2V→2.075V（理想 2.1V），跨 3.0–4.2V 全程 **−25mV 节点偏移 = −50mV 电池读数偏移**（50nA 漏电假设）。
- 源阻抗 500kΩ；瞬态 τ=R∥×C=500k×0.01µF=**5ms**，~25ms 稳定 → 对慢变电池电压 OK，但放大漏电误差。
- 结论：分压**比例正确**，问题在**绝对阻抗偏高**（→ HW-P1-A）。

**④ 硬件↔固件交叉**：见第五节。

## 四、待实测清单（上板/datasheet 确认，不下定论）

1. **HW-P1-A**：万用表实测 VBAT_DET 节点电压 vs 电芯电压，核 ADC 实际偏差是否在可接受范围；评估是否把分压改 200k//200k（源阻抗降到 100k）。
2. **HW-P2-B**：充电时实测 TP4054 表面温度 + 充电电流；确认电芯规格书允许 0.45C。
3. **HW-P3-E**：确认 40MHz 晶振 CL 规格、C728 实际容值、L644 设计意图（是否给 4G 参考时钟）。
4. **HW-P2-D**：确认 NFC 是否轮询模式；如需中断需补 WS_IRQ → MCU GPIO。
5. I2C codec(U62) 实际地址，与 NFC 是否冲突。

## 五、硬件↔固件交叉矩阵（最高价值）

| 主题 | 硬件实测 | 固件假设 | 结论 |
|---|---|---|---|
| 电池分压比 | R250/R687 = **÷2** | `voltage*2` | ✅ 一致，分压比无误 |
| ADC 满量程 | — | 回退 Vref=3600mV | ❌ **08-P0-A**：3600 偏高约16%→该关机不关→过放（头号红线，固件侧） |
| ADC 源阻抗 | **500kΩ 偏高** | 未补偿高阻抗偏差 | ⚠️ HW-P1-A：−50mV 偏低，方向与 P0-A 相反但量级小，不抵消 |
| 充电状态 | TP4054 **单 CHRG 脚** | 单 GPIO 读充电 | ✅ 固件正确，但 08-P2-E"区分满充"**硬件做不到**，需加硬件 STDBY 脚或纯固件超时降级 |
| ADC 通道 | BATTERY_ADC=CH7 | 硬编码 CH7 | ⚠️ 08-P1-B：当前巧合一致，改板会读错脚 |

**一句话总结**：硬件本身没有 P0 级死穴，分压比/复位/充电拓扑都对；**真正卡量产的电源红线在固件 08-P0-A（Vref 高估→过放）**，外加硬件 HW-P1-A（分压源阻抗偏高）会在未校准 + 高阻抗叠加时让电量读数更不可信。出厂强制 ADC eFuse 校准 + 评估降低分压阻抗，是性价比最高的两步。
