## 0. 实施顺序与前置依赖（implementationOrder）

```
里程碑 A（共享基础，必须最先做）
   └─ protocol_ unique_ptr → shared_ptr 改造（声明 + 所有 assign/reset/本地拷贝惯例）
         ↓ 是 R2 / R3 的强前置（不先做则 sender/reconnect 任务 = 偶发 UAF）

里程碑 B（独立，可与 A 并行或先于 A，互不依赖）
   ├─ R1  运行期过放保护闭环（电源红线，独立）
   └─ S2  唤醒词编码任务 UAF/重入（并发红线，独立，custom + afe 同改）

里程碑 C（依赖 A 落地，R2+R3 必须合并设计/实测）
   ├─ R2  AudioSender 上行任务 + 控制命令异步化 + WebSocket 并发加固
   └─ R3  可取消重连任务 + Reboot/ResetProtocol join + 全量 protocol_-> 走本地拷贝
         ↑ R2/R3 共享 AT 锁争用、protocol_ 生命周期、主循环剥离，合并 plan、合并真机实测
```

**强约束：**
- A 必须先于 C 合入。C 的所有跨任务 protocol_ 解引用都建立在 A 的 `atomic_load` 本地拷贝惯例之上。
- R1、S2 不依赖 A，可作为独立里程碑先行评审/实测，降低 C 的批量风险。
- R2 与 R3 **不可拆开单独合入**：二者共享 AT 锁优先级、reconnect/sender 任务 join 入口、protocol_ 本地拷贝清单，必须同一里程碑、合并真机弱网+断网+打断实测。

## 1. 共享基础改造先做项（crossCutting）

这些是 R2/R3 共同依赖、必须在动任意任务抽离之前完成的结构性改造，全部归入**里程碑 A**：

### A-1 · protocol_ 改 shared_ptr + atomic_load/store 惯例

**声明（application.h:158）**
`std::unique_ptr<Protocol> protocol_;` → `std::shared_ptr<Protocol> protocol_;`（`<memory>` 已在，无需引头文件）。C++17 用自由函数 `std::atomic_load/atomic_store(&protocol_, …)` 做原子读写，不引入额外 mutex。

**所有 assign 改（InitializeProtocol :663/669/672/674/678）**
`make_unique` → `make_shared`，先赋给**局部** `std::shared_ptr<Protocol> p`，在 `p` 上注册全部回调（:681-873）+ `SetFrameDuration(p->client_frame_duration())`（:876），最后一步 `std::atomic_store(&protocol_, p);` 发布，再 `p->Start();`。**先注册后发布**，避免别的任务 load 到回调未注册的半成品。

**所有 reset 改（Reboot :1363 / ResetProtocol :1558）**
`protocol_.reset();` → `std::atomic_store(&protocol_, std::shared_ptr<Protocol>{});`。调用任务只释放自己那份引用计数；若 sender/reconnect 任务正持本地拷贝，对象延后到它们用完才析构 → 杜绝 UAF。

### A-2 · 全量 protocol_-> 裸解引用改本地拷贝惯例（R3-H1 补强：必须逐点，不得"基础方案统一"一笔带过）

**统一惯例：任何解引用 protocol_ 的函数，开头先取本地拷贝，全程只用局部变量：**
```cpp
auto p = std::atomic_load(&protocol_);
if (!p) { return / continue; }
p->Xxx();   // 整段只用 p，不再二次读成员 protocol_
```

### A-3 · 风险与边界

- **shared_ptr 只挡 UAF，不挡"任务仍在跑"**：reset 后对象延后析构，但 sender/reconnect 任务可能仍持旧 p 跑完一轮长阻塞 I/O（最长 ~10s）。表现为关机/切网延迟数秒，非崩溃。须由 R2/R3 的 **cancel + join** 配套（见 C 各项），A 单独合入功能完整但不消除关机延迟。
- **atomic 自由函数 vs 裸访问混用 = data race**：全量替换无遗漏（A-2 清单已穷举）。C++17 这些自由函数已 deprecated，工具链仍可用，编译可能出 deprecation warning，非错误。
- **A 可单独评审先行**：单独合 A 不改变现有同步任务行为（主任务内仍串行安全），作为 R2/R3 的前置基础先评审。

- **异常**：(a) 降 3.39V 后立即回升 3.46V(>3450) → streak 清零不关机；(b) 3.41V 带内横跳 10min → streak 不增不清，无关机日志。
- **边界**：(c) 恰 3.40V(=3400 非<3400) → 不累积；3.399V → 累积关机（验严格小于）；(d) 关机倒计时(3000ms)内插充电器 → Schedule 二次确认 IsCharging=true，打"取消"日志不 deep sleep；(e) 缺陷2 二次触发：3.39V 触发→倒计时内插充→拔出→再降 3.39V 持续 180s → 必须二次关机（验 ResetOffBatteryGuard 清 fired_）；(f) 缺陷1 开机即过放：先设 3.39V 再上电 → 正常启动不崩（ready_+fix1·B 抑制），运行期再走 180s 关机；(g) fix3：关机倒计时内触发第二个关机源（手动唤醒再见关机）→ 第二次被哨兵吞掉，无 AudioService 二次 Stop。
- **压力**：(h) 3.39V + 弱网（主循环被占）→ Schedule 不丢失最终关机；**fix4：R2 未落地前过放关机最坏端到端 = 180s + 主循环最大阻塞时长，实测记录该阻塞峰值作为"R1 能否先于 R2 单独出货"门禁**（若弱网排队 > 电芯过放安全窗口，R1 须与 R2 同里程碑）；(i) 上电/断电 ADC 死值不误关（复用 sc7a20h 经验）；(j) 4G PA 发射时抓 Cali Voltage 波动 <50mV。

- **边界**：① 3 次全失败 → 干净回 Idle、清屏、reconnect_running_=false；② 5min 窗口第 4 次断线 → window 限流不起重连任务；③ user/server_initiated_close 早退 → 不进重连、不计数（F3 验证）。
- **H2 专项（必加）**：(a) ResetProtocol(切网)期间测量主循环冻结时长——验证 join 在板级线程而非主任务自旋（CLOCK_TICK 不冻结）；(b) **重连任务卡 AT WaitBits 时拔 4G 模组（非拔天线）+ ResetProtocol**：验任务能退出、protocol_ 引用最终释放、uxTaskGetSystemState 无僵尸 reconnect（F2-c）。
- **压力**：MIPOPEN/WS握手/hello 三种超时分支各跑；连续断-连 30min 老化：minimum_free_size 无单调下降（采样 60s，末值≥首值-2KB）、uxTaskGetStackHighWaterMark 标定 reconnect 栈水位收敛。
- **TTS/教育卡异步专项**（若 F6 定为异步）：弱网触发 TTS/NFC 切 prompt → 主循环不卡（CLOCK_TICK 1s±100ms），开通道后正常发声/切 prompt；失败干净回退不残留 Connecting。

**redline=true · planModeRequired=true · 依赖：里程碑 A；与 R2 合并里程碑 C**

## 6. 总实测门禁（testGate · 合入标准）

任一红线项合入前，对应里程碑必须在真机（开 heap poisoning comprehensive + assert + FREERTOS_CHECK_STACKOVERFLOW）通过下列量化门禁：

| 维度 | 门禁阈值 |
|---|---|
| 主循环实时性 | 弱网/重连/关机全程 CLOCK_TICK 间隔 **1s ± 100ms**；触摸到状态变化 **<200ms**；主循环无 >200ms 停顿 |
| 打断延迟（R2） | 弱网对话中打断 **P95 < 500ms**，30 次无失败（AT 锁抢占必做才达标） |
| 内存稳定 | 30min 弱网+反复断网重连老化：`heap_caps_get_minimum_free_size` 无单调下降（采样 60s，末值 ≥ 首值 − 2KB）；S2 PSRAM free 前后差 <8KB |
| 任务栈 | 新增 sender/reconnect 任务 `uxTaskGetStackHighWaterMark` 标定，余量 **≥1KB** |
| 无 UAF | 重连/sender 卡 10s WaitBits 时 Reboot+切网 ×100，heap poisoning+assert：**0 UAF、0 assert、0 watchdog（TWDT）、0 CORRUPT HEAP** |
| 无僵尸任务 | 拔 4G 模组 + ResetProtocol 后 `uxTaskGetSystemState` 无残留 reconnect；切网路径 protocol_ 引用最终释放 |
| 通话维持（产品取舍） | 弱网 3min 持续对话 **不断开重连**，仅 audio_drop_count 单调增、无重启 |
| 过放关机（R1） | 3.39V 持续 → 首次<3400 到 deep sleep **≤约240s**；迟滞/二次触发/充电取消/开机即过放/多关机源哨兵 全部按预期 |
| 跨板型一致 | R1/S2 两板型(4g/wifi)关键用例最小集各跑一遍，行为一致 |
| WiFi 并发对照（R2） | WiFi 路径同跑弱网+打断+heap poisoning，WebSocket continuation_/send_mutex_ 改造无 WS 帧损坏 |

合入流程：先 plan → Jack 双确认 → 退出 plan mode 实施 → 上述门禁真机实测通过 → 合入。**不在普通模式直接改电源/网络主链路/唤醒词文件。**
