# P30 压力测试方案（针对本轮 P0/P1 修复的回归与量产验证）

> 目的：验证 p30 分支上的 20 个 P0 全清 + 本轮 P1 修复，并按量产门禁做压测/老化。
> 适用：P30-4G、P30-WiFi 两个板型。建议各测一遍；4G 专项只在 P30-4G 上做。
> 全程开串口日志录制；每个用例标注【验证】对应的修复项编号，便于追溯。

---

## 〇、前置：冒烟回归（必须先过）

1. 两个板型各 `idf.py fullclean && idf.py build` 编译通过、烧录开机。
2. 基础功能走一遍：开机→联网/配网→唤醒→对话→TTS→打断→息屏→充电，无崩溃、无卡死。
3. 串口确认开机后可用内部 RAM > 60KB（`heap_caps_get_free_size(MALLOC_CAP_INTERNAL)`）。

---

## 一、电源域压测（本轮重点）

| 用例 | 步骤 | 预期 | 验证 |
|---|---|---|---|
| 未校准芯片电量 | 用串口打印 `eFuse not burnt` 的样机开机 | 电量显示正常(满电~4.1V/接近100%)，**不**开机即关机 | 05b-P0-A、05a-P0-A1 |
| ADC 抖动不重启 | 充电口反复快速插拔 200 次 | 不出现随机重启/panic | 05-P0-1 |
| 过放保护 | 持续放电到自动关机 | 5% 时有低电告警，约 3.4V 关机不再过放 | 05a-P0-A1 |
| 背光压测 | 自动调光/进出省电下连续来回调亮度 500 次 | 亮度过渡平滑，不跳变、不卡死 | 05a-P1-A2 |
| 充电状态翻转 | 反复插拔充电 + 观察电量/充电图标 | 图标/电量不乱跳、不崩 | 05-P1-* |

## 七、长稳 / 老化压测（量产门禁）

1. **72 小时连续运行**：正常使用循环（唤醒/对话/息屏/充电）自动化跑 72h。
2. **内存监控**：每 10s 记录可用内部 RAM、最大空闲块、各任务栈高水位；全程内部 RAM **不得跌破 60KB**，无单调下降（泄漏）。
3. **温度老化**：高温(如 45℃)环境跑 24h，重点观察电源/充电/4G。
4. **看门狗**：全程无 WDT 复位；记录任何 panic/coredump。

---

## 八、全程监控指标与通过标准（出货门禁）

**监控**：崩溃/重启次数、coredump、可用内部 RAM 曲线、堆碎片、任务高水位、关键日志（adc read failed / invalid index / 模组未检出 等出现即记录但不应导致崩溃）。

**通过标准**：
- P0 相关用例：**零崩溃、零砖机**（必须）。
- P1 相关用例：无高频崩溃；偶发问题记录并评估。
- 72h 老化：无内存泄漏、无 WDT、内部 RAM 不破 60KB。
- 配网/4G/打断：成功率 ≥ 约定阈值（建议 ≥99%）。

任一 P0 用例出现崩溃/砖机 → **不予放行**，回到对应修复项复查。
