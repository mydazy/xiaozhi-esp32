# 触摸乱跳现场复查（P30-4G / AXS5106L）— 2026-06-07

> 现场症状：触摸【用一会儿就乱跳】（运行期累积，非开机即有）+【发热后更明显】（温度相关）+【仅 4G 批次】（WiFi 批次无）。
> 复查范围：以当前分支代码为准（main/ components/），只读不改。证据标 文件:行号。
> 板型：P30-4G。触摸 IC = AXS5106L（电容自电容，I2C，地址 0x63）。4G 模组 = ML307R。

---

## 一、最可能根因链（解释三特征）

**根因不在固件，在硬件级 RF/电源耦合；固件只能缓解不能根治。**

四维度 + 对抗验证后，候选嫌疑收敛如下（按可信度排序）：

| 嫌疑 | 验证结论 | whoFixes |
|---|---|---|
| 4G 主天线 A2/A3 RF 近场耦合进 AXS5106L 自电容感应线/FPC | **存疑（未证伪，物理自洽，需真机近场实测）** | 硬件根因 |
| 4G RF 耦合到触摸 INT 线（GPIO5）产生伪边沿 | 间接成立（驱动整套抗扰建模即针对此） | 软硬协同 |
| 4G RF 耦合到 I2C(GPIO11/12) 数据位翻转→脏坐标帧 | 存疑（物理可能，代码侧不可确证） | 软硬协同 |
| 无坐标层滤波/逐帧剔除（透传放大噪声） | 存疑（非根因，是有效缓解杠杆） | 固件可缓解 |
| 无运行期热重校准（基线随温漂无兜底） | 存疑（事实成立，热 vs RF 因果未决，需厂商确认 V2907） | 软硬协同 |
| **read_register 退回非原子 write+read（072cd457 回归）** | **证伪**（I2C 跨地址隔离，读不错别设备数据） | — |
| 4G 大电流污染 AUD_VDD-3.3V 轨→供电纹波 | **证伪**（PA 大电流走 VBAT 不走 GPIO9 轨，HARDWARE.md 拓扑） | — |
| 片内温度传感器未接触摸补偿 | **证伪**（两批次共有缺失，不能产生 4G 独占症状） | — |
| storm 活跃期被跳过=盲点 | **证伪**（storm 只管 INT 边沿不碰坐标值，按压期另有速度/范围门） | — |
| active-press 放行 INT debounce | **证伪**（INT 门只判存在性不读坐标值） | — |

**根因链（最可能）：**
ML307R 4G 发射突发（TDD/PUSCH，仅 4G 批次有此源）→ RF 近场 + INT 边沿伪触发 + 可能的 I2C 位翻转，同时注入触摸通道 → 被 RF 污染但落在屏内合法范围的坐标穿过固件全部校验（速度门只挡快速大位移、范围门只挡越界）→ UI 乱跳。
- **仅 4G**：WiFi 批次无 ML307R，无此 RF/发射源。这是三特征里最硬的区分点，直接排除一切"两批次共有缺失"类嫌疑。
- **运行期累积**：必须有人在用（pressed=true）+ 发射活动叠加才暴露，开机冷态空闲不触发。
- **发热后更明显**：温升 → PA 增益压缩、闭环功控抬高发射电流/场强 → RF 注入更强；叠加电容触摸自基线随温漂使信噪比下降、抗扰余量被吃掉。这两条都是物理量随温度变化，**纯固件参数无法消除注入源**。

**为何确信非固件 bug**：驱动已堆一整套抗扰（INT 三连电平确认、storm 边沿检测、双档 RF 参数、post-release guard、速度上限、单帧伪 press 拦截、I2C bus_reset+retry），commit 历史一版版反复调参（debounce 4→6ms、storm 阈值 18→14、speed 2500→2000，b603bcbb 自述"还是会干扰"）。软件能压住的早压住了——压不住、仅 4G、越热越明显，是硬件耦合的典型特征。

## 二、固件侧可做（firmwareFixes）

> 全部为**缓解**，无一可根治（根治需硬件去耦/屏蔽）。按性价比排序。

### F1【缓解·最高性价比】坐标层加中值/逐帧跳变剔除
- **现状缺口**：`read_touch`（axs5106l_touch.c:649-717）对单帧坐标只做 clamp（698-701）+ 一个速度上限（703-710，STRICT 档 2000px/s）。无中值滤波、无滑动平均、无多帧一致性比对。`touch_state_t`（86-96）只存一帧 last_x/last_y。lvgl_read_cb 把单帧 x/y 直接写 data->point（593-595）。
- **改动点**：在 read_touch 输出前加 3 点中值（需在 touch_state_t 扩 2 帧历史缓冲）+ 邻帧曼哈顿距离硬门限（独立于速度门，挡"离散瞬移"而非仅"超速连续移动"）。
- **效果**：压制 RF 注入的小幅高频抖动与单帧离散瞬移；对"慢速小幅乱跳"（现速度门拦不住，2000px/s×30ms≈60px 容差）有效。

### F2【缓解】运行期热/周期重校准
- **现状缺口**：`write_register` 全文仅 2 处——SLEEP(0x19, :365)、RESET(0xF0, :457)。运行期从不下发重校准。唯一能重建基线的 `reset_chip`(:453-466) 只在 init/resume 调；`axs5106l_touch_resume()` 头注释明写"当前三板未用"（axs5106l_touch.h:97），main 侧 grep 证实运行期不调。
- **改动点**：加周期定时器（如每 N 分钟）或温升触发，运行期在"确认无触摸 + 空闲"窗口下发 reset_chip/重校准命令重建基线。
- **前置依赖（待确认）**：V2907 firmware 是否暴露可写的重校准/charger-mode/noise-mode 寄存器——需厂商寄存器手册。若仅 RESET 重建基线，运行期复位会打断交互，需谨慎窗口。**这条能否真缓解取决于厂商，标软硬协同。**

### F3【缓解·若硬件能给信号】4G 发射期主动屏蔽/重读
- **现状缺口**：触摸纯 30ms 轮询（axs5106l_touch.h:87），与 4G TX 完全异步。`OnNetworkStateChanged`（ml307_board.cc:95）只传 network_ready（连接态），无 TX-active 回调。touch 驱动无 set_tx_active/mute_on_tx 入口。固件不知道"此刻在发射"。
- **改动点**：若硬件能引一根 ML307R TX-active 状态 GPIO（或 AT 上报 TX 窗口），touch 驱动在发射窗内 mute/丢弃/重读坐标。
- **限制**：TX 突发仅几百 μs~几 ms，跨不过两个 30ms 轮询，多帧确认本已部分接住；但发射期主动屏蔽能根除"发射瞬间脏帧进 UI"。**依赖硬件给信号，标软硬协同。**

### F4【缓解·已在做但未尽】继续收紧速度门/抖动门，不要误以为已修
- speed 门（:709）+ jitter（JITTER_LIMIT_FOR_TAP=40, :54）+ 范围门（:676-678）对 pressed 帧全部生效，是当前主防线。
- **排查盲区警示**：commit fc53cfcb 的 err_streak 缓解（i2c_bus_worker.c:213-224，仅 `ret!=ESP_OK` 才 err_streak++）只防"通信失败→bus_reset 风暴"，**不治"通信成功但坐标被 RF 污染"**。RF 翻位但 I2C 仍正常 ACK 时返回 ESP_OK，err_streak 归零，整条 reset 链不触发，坐标已脏却透明。日志看 i2c=0、无 bus_reset 会误判"I2C 没问题"。**不要把 fc53cfcb 当成已修触摸乱跳。**（i2c_bus_worker.h:53-58 注释自陈"bus_reset 无法修复设备层"）

### 不必做（对抗验证已证伪，避免无效改动）
- ~~把 read_register 改回原子 i2c_worker_write_read~~：072cd457 确实改回了非原子（axs5106l_touch.c:486-487 vs 62541b9d），但 I2C 每条 op 自带从机地址寻址（i2c_bus_worker.c:126），跨地址隔离，插队读不错别设备数据，最多影响时延/超时。**改回原子可恢复一致性（无害），但它不是乱跳根因，别指望它修乱跳。**
- ~~storm 活跃期补防护~~：storm 只统计 INT 边沿、命中即整帧 RELEASED，不碰坐标值；按压期坐标防线本就是速度/范围/抖动门，盲点不存在。
- ~~die 温度接触摸补偿~~：两批次共有缺失，逻辑上不能产生 4G 独占症状；且 die 温度≠触摸 IC 温度，非合格补偿输入。

## 三、硬件根因嫌疑（需硬件团队 + 示波器/频谱仪验证）

> 共享证据：config.h:30 — ML307R + 音频 + LCD 三者共用 GPIO9（AUD_VDD-3.3V）；config.h I2C 上拉外置（mydazy_p30_board.cc:170 enable_internal_pullup=0）。

### H1【高】4G 天线 A2/A3 RF 近场直接耦合 AXS5106L 自电容
- AXS5106L 焊于屏 FPC（HARDWARE.md:140），与 4G IPEX 主/辅天线 A2/A3（HARDWARE.md:173）同板近距。电容触摸测 fF 级变化，对 RF 近场极敏感。
- **排查**：近场探头测 A2/A3 区域 RF 场强 vs 触摸 FPC 位置耦合量；发射期（连 PPP/上传时）对比触摸坐标抖动。
- **缓解方向**：触摸 FPC 加屏蔽层/接地铜箔、天线远离触摸感应区、感应线包地。

### H2【中-高】4G RF 耦合到 INT 线(GPIO5)产生伪边沿
- 驱动整套抗扰（storm 检测、INT 三连确认、双档 debounce）即针对此路径建模。b603bcbb"还是会干扰"说明 INT 层缓解未尽。
- **排查**：示波器抓 INT(GPIO5) 在发射期的伪边沿；与 storm 日志比对。
- **缓解方向**：INT 线串 RC 滤波、加上拉、走线远离天线。

### H3【中】4G RF 耦合到 I2C(GPIO11/12) 数据位翻转
- 触摸与音频共用 I2C1（config.h），4G RF 耦合到 SDA/SCL 造成位翻转，read_touch 读回 6 字节帧被污染。落在合法范围的位翻转固件拦不住。
- **排查**：示波器看 I2C 波形在发射期完整性；统计 ESP_OK 但坐标异常的帧率。
- **缓解方向**：触摸迁到独立 I2C_NUM_0、I2C 走线包地、上拉值优化。

### H4【中】触摸供电去耦/上拉随温漂
- I2C 关内部上拉依赖外部上拉（mydazy_p30_board.cc:170），发热后上拉电阻/PCB 寄生 RC 漂移使 SDA/SCL 边沿变缓。AXS 无独立 LDO，挂共享轨。
- **排查**：热成像看发热后哪个器件温升最大（4G PA？LDO？）；示波器看温升后 I2C 边沿斜率/触摸 VDD 纹波恶化。
- **缓解方向**：开内部上拉或换外部上拉选型、AXS VDD 就近加去耦电容、触摸独立 LDO。

> **已证伪的硬件假设（别查偏）**：4G PA 大电流污染 AUD_VDD-3.3V 轨——PA 主电流走 VBAT（直连电池，HARDWARE.md:440/453），GPIO9 只切 ML307R 的 VDD_EXT 非主供电，大电流不流经触摸所在轨。

## 四、复现验证（坐实归因）

三组对照实验，目标是把"仅4G+热+乱跳"三特征逐一锁到物理源：

### 实验 A：4G 发射期 vs 静默期（锁定"4G 发射是注入源"）
- 设备触屏持续操作，分两段：① 4G 模组飞行模式/断网（无发射）；② 强制持续上行（大文件上传/ping flood，PUSCH 满占空）。
- **判据**：若仅 ② 段乱跳显著、① 段正常 → 坐实发射是注入源。同步示波器抓 INT(GPIO5)/I2C(GPIO11/12) 在 ② 段的伪边沿/位翻转。

### 实验 B：发热箱升温（锁定"温度相关机制"）
- 发热箱阶梯升温（常温→40→55→70°C），每档跑实验 A 的 ② 段，记录单位时间乱跳次数 + 热成像。
- **判据**：乱跳率随温度单调上升 → 坐实热相关。热成像定位最热器件（PA/LDO/触摸 IC）→ 区分是"PA 功控抬升场强"还是"触摸基线温漂"主导。

### 实验 C：对照 WiFi 批次（锁定"4G 独占"）
- 同固件版本（确认 WiFi 板用 RF_NORMAL、4G 板用 RF_STRICT，mydazy_p30_board.cc:334/339），WiFi 批次跑实验 A+B 同样流程。
- **判据**：WiFi 批次在任何温度/发射状态下均不乱跳 → 排除一切两批次共有的固件/算法因素，坐定差异源 = ML307R 4G 发射。

### 厂商确认（裁决 F2 热重校准是否有效）
- 向 AXS5106L 厂商确认 V2907 firmware 是否内置自动温度重校准及触发条件、是否暴露可写重校准寄存器。决定 F2 能否做、能做到什么程度。

## 五、优先级（固件先做 vs 硬件待办）

**固件先做（快缓解，本周可落地，不依赖硬件改板）：**
1. **F1 坐标中值滤波 + 逐帧跳变剔除**（性价比最高，纯软件，直接压小幅抖动/离散瞬移）。
2. **F4 速度/抖动门微调 + 打掉"fc53cfcb 已修"误判**（确保排查不被假象误导）。

**待硬件信号/确认后做（软硬协同）：**
3. **F3 4G 发射期屏蔽重读**——需硬件引 TX-active GPIO。
4. **F2 运行期热重校准**——需厂商确认 V2907 寄存器。

**硬件待办（治本，必须真机+仪器）：**
5. 跑实验 A/B/C 坐实归因 → 按结果做 H1（FPC 屏蔽/天线隔离，首选嫌疑）/ H2（INT 滤波）/ H3（I2C 迁总线+包地）/ H4（去耦+上拉）。

> 关键认知：固件 F1~F4 全是缓解，能让乱跳变稀但消不掉。**根治必须硬件去耦/屏蔽**。别再单纯调 RF 参数档反复试（已证明收敛不掉）。

## 六、四句话回报（fourLines）

1. **做了什么**：复查触摸乱跳，确认根因是 4G 模组发射的 RF/电流耦合干扰触摸（硬件问题），不是固件 bug；上一轮被证伪的"I2C 读法回归""storm 盲点"等代码嫌疑这次也复核确认不是真因。
2. **影响什么**：只影响 4G 批次的触摸手感（乱跳），WiFi 批次不受影响；不涉及安全/砖机。固件层能做的几个滤波/重校准只能让它变稀，消不掉。
3. **要不要你拍板**：固件侧我先上"坐标滤波+异常剔除"（纯软件、低风险、本周出），能明显减轻乱跳；但要彻底解决，需要硬件团队上仪器查（天线屏蔽/I2C 走线/供电去耦），这步必须真机实验。是否同意"固件先缓解、硬件并行排查"两条腿走？
4. **下一步**：① 我做固件滤波缓解；② 硬件团队按实验 A/B/C（发射期对照 + 发热箱 + 对照 WiFi 批）上示波器坐实是哪条耦合路径；③ 同步找 AXS 厂商确认 V2907 能否运行期重校准。
