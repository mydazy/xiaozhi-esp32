## 四、出货前必清零清单

**硬 ship-blocker（4 条）：**
1. **09-P0-D** `at_uart.cc:553` — `cme_error_code_ = arguments.empty()?0:arguments[0].int_value;` 一行守卫。验证：`+CME ERROR:`（空码）不崩。
2. **sdkconfig 量产态** — 改 `sdkconfig.defaults` 把日志回 ERROR 级（`MAXIMUM_EQUALS_DEFAULT`）、关 `RUNTIME_MONITOR`，reconfigure 重出固件。
3. **陀螺仪休眠唤醒配套防误触**（4G+WiFi）— **【Jack 2026-06-07 决策：有意恢复"拿起唤醒/摇一摇"，走配套方案，不回退】**。裸开 `true` 不可出货，必须配套以下四点，把"桌面震动误开机+大声播报"压住：
   - ① **EXT1 唤醒静音启动**：唤醒后判定来源为 EXT1(陀螺仪) 时，低音量/静音启动，**不播"联网成功"等提示音**（直击 G1 半夜大声播报）。
   - ② **震动 vs 真拿起区分**：唤醒后短窗口做姿态二次确认（持续抬起/角度变化/连续运动），桌面单次震动不满足→立即回深睡，不进整机启动。
   - ③ **时段/状态门控**：仅非充电静置时段 arm EXT1；可选夜间时段不 arm（直击 E1/E3 整夜反复开机+耗电）。
   - ④ **EXT1 灵敏度阈值真机标定**：把 `sc7a20h_wakeup` 的触发门调高到"真拿起才触发"。
   【红线：动休眠/电源域，需 plan + 双确认 + 真机实测；①②③ 是代码，④ 是真机调参】
4. **08-P0-A 电源校准** — 建出厂校准流程（烧 eFuse 或标定写 NVS 真 Vref），未校准兜底从 3600 降到实测真值（约 3100）。【红线 + 真机双确认】

**有条件项（1 条）：**
5. **D1 通话保活唤醒** — 真机 realtime 通话 + 喊唤醒词打断，开 RUNTIME_MONITOR 看 internal min_free 不破 60KB，30min 老化。注释自定的合入门禁，未做不放行。

**降级 P1（可放行，下里程碑必补）：**
- 07-P0-A 读端 40 处补 `auto p = std::atomic_load(&protocol_)` 本地拷贝。当前因 SwitchProtocol 已删、必崩路径消失而暂安全——但这层保护没写进代码，谁加回平台切换就原地复发 P0。

**决策项（交 Jack 威胁模型，非阻塞）：**
- 04-P0-A OTA 验签/Secure Boot（首版可后放，建议下载 URL 强制 https）；04-P0-B 远程命令鉴权（降 P1，危险命令加白名单）。
