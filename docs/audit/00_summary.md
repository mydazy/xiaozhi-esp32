## 四、复核重要结论（来自 99_review.md，必读）

1. **CLAUDE.md 的"头号待修 05b-P0-A 漏乘分压系数"经实地核实不成立** —— 两条换算分支都有 `*2`，没漏乘。真问题是 **08-P0-A：回退参考电压 3600mV 偏高 + 出厂校准缺位**。请以本轮 §三 与 `99_review §2.1` 为准，修订对 CLAUDE.md「七、当前状态」一节的描述（该文件改动属指令文件，留待你确认）。
2. **03-P0-G（TLS 中间人）已降级 P1**：底层 `esp_ssl.cc:38` 已开 `crt_bundle_attach`，CA 校验默认开，原 P0 前提不成立。
3. **8 处漏报盲区待补审**：充电截止逻辑未核实、休眠唤醒源↔状态机冷启动衔接、blufi 超长 SSID 流向 wifi_config 下游、资产解析链未纳入 OTA 安全治理、i2c worker 全局抢占、`web_socket.cc:134` stoi 漏网等。详见 `99_review §3`。
4. **子系统交界无真空**：08/09（board.cc 电源 vs 外设切片）、01/02、03/07 三处经核实各自划线到位。

## 五、量产门禁状态

当前处于「全量审计完成」阶段。下一步：**P0 清零 → P1 收敛 → 小批试产实测（重点电源/温度/老化）→ 量产**。

P0 清零建议优先序（离量产近、风险高、改动小优先）：
1. **09-P0-A/B/C/D 一组**（4G 脏帧越界）：均一行守卫即修，弱网现场必现，性价比最高。
2. **08-P0-A**（电源 Vref/校准）：电池红线，整批过放风险，需配出厂校准流程一起定。
3. **04-P0-A + 04-P0-B**（OTA 无验签 + 远程零鉴权）：唯一可被远程批量砖机的组合，安全红线。
4. 其余 8 条按子系统排期，与同源 P1（GLB 主题）一并整改。

> 注：以上均为软件层 + 配置层缺陷分析，未对任何代码做改动。涉及电源域/OTA/分区表的整改属硬红线，须先列清单 + 实测双确认后再动手。
