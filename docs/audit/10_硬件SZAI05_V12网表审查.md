# 10 — 硬件 SZAI05 V12 网表审查（KiCad 转换 + 固件对照）

> 日期：2026-06-11 ｜ 数据源：`SZAI05_V12_20251231end.brd`（Allegro 16.6）经 KiCad 10 官方导入器转换，网表 304 网络 / 1004 引脚与源文件比对 100% 一致。KiCad 文件位于 iCloud `…/MyDazy-P30/KiCad_P30_V12/`。

## 一、最重要结论：这块板是 P31（新版），不是 P30-4G

证据链（三条独立证据互相印证 `mydazy-p30-4g/README.md` 的 P31 对比表）：

| README P31 特征 | 网表实证 |
|---|---|
| 音频 DAC = ES7111（纯 I2S，无 I2C） | U75：I2S_DSDIN1/SCLK1/LRCK1 输入，OUTP/OUTN 输出，无 I2C 连接 ✓ |
| 充电检测 = GPIO_44 | TP4054 CHRG(U56.1) → R570 → U0_RXD 网络（U53.50 = GPIO44）✓ |
| Type-C 耳机 | MICBIAS_USB / UMICP / UMICN / USB_MIC_ADC(GPIO15) / MIC_SELECT(GPIO21→U74 模拟开关) 全套 USB 麦克风电路 ✓ |

另：4G 模组为 ML307C（非 P30-4G 老图的 Air780EP），LCD/触摸为 AXS5106L（U61）。

**影响**：当前 `main/boards/mydazy-p30-4g/` 固件直接烧这块板，至少两处必坏：
1. **音频放音失败**：固件 BoxAudioCodec 走 ES8311 I2C 初始化，板上无 ES8311，I2C 探测失败（板上 DAC 是 ES7111，需 Es7111AudioCodec 驱动）。
2. **充电检测错位**：固件读 GPIO21（POWER_MANAGER_GPIO），板上 GPIO21 是 MIC_SELECT（麦克风切换输出脚）——读出来的不是充电状态，且若固件把它配成输入，Type-C 耳机麦克风切换会失效。充电检测应在 GPIO44。

**仓库尚无 `mydazy-p31` 板型目录**，README 只有对比表。需要新建板型（任务建议见 §四）。

## 二、电源域（对照固件 08-P0-A）

- **电池分压**：VBAT_4V2 → R250(1M 1%) → VBAT_DET(U53.13 = GPIO8, ADC1_CH7) → R687(1M 1%) → GND，C350=10nF 滤波。比例 2:1，与固件 `*2` 换算一致 ✓（99_review §2.1 的证伪结论再次实证）。
- **注意**：1M+1M 源阻抗约 500kΩ，远超 ESP32-S3 ADC 推荐输入阻抗，靠 10nF 储能；要求固件采样间隔足够（连续快采会拉低读数）。这会叠加 08-P0-A 的 Vref 误差——**出厂校准方案设计时两者要一起标定**（校准点覆盖真实采样时序）。
- 充电：TP4054(U56)，VIN→充电→VBAT_4V2；Q15 功率路径切换。
- 4G 供电：GPIO9(AUDPWR_EN, U53.14) → VT7 → Q38(PMOS) → ML307 VBAT(U66.42/43)，与固件"音频+4G+LCD 共享电源开关"注释一致 ✓。固件靠断电硬复位 4G 的设计在硬件上成立。
- LDO：ETA5050V33S2F ×2（U55/U58），ME6211 未在 V12 出现（老图器件）。
