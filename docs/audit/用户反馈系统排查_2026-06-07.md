## 一、最痛根因与全局图景（overview）

现场 18 条反馈收敛为 **6 个真根因簇**，按"是否半夜扰民 / 是否伤电芯 / 是否首触即坏"排序，**头号是 E+G 电源·网络扰民簇**，不是音量也不是触屏：

1. **【头号·E+G 簇】陀螺仪唤醒(EXT1)在息屏休眠时恒开 → 整夜自动开机 / 联网播报扰民 / 拖垮续航**。
   单一根因：`mydazy_p30_board.cc:511` 的 5 分钟无操作休眠走 `ShutdownOrSleep(...,enable_gyro_wakeup=true)`，深睡时 arm 陀螺仪 EXT1 唤醒源（`:595 ArmGyroWakeup`）。夜间桌面震动/碰一下/搬动 → 陀螺仪触发 EXT1 → 整机醒来重启 → boot 后 4G 重连播 `OGG_NETWORK_4G`"联网成功"（`dual_network_board.cc:51`）按 80% 默认音量大声播 → **同一个根因同时解释 E1（整夜自动开机 4 次）、E2（息屏自动开机/拿起需长按）、G1（夜间大声播联网成功吵醒全家）、E3（反复醒来重连拖垮续航）**。冯工现场答"靠近自动对话新功能所致"= 就是这个陀螺仪 shake-wake。**唯一 gyro 关掉的路径是 4 连击"再见"关机（`:783 enable_gyro_wakeup=false`），但用户不知道要四连击，自然走休眠 → 必现。**

2. **【C 簇·已坐实高置信】低音量被开机逻辑反向覆盖**。`ApplyDefaultSettings()` 每次开机"音量<50 强制回写 80"（4g `:857-862` / wifi `:800-805`），切网=`esp_restart`重启同源 → 精确匹配现场"≤40%不持久化、≥50%保留"。这又放大了 G1：每次半夜重启把音量拉回 80% 大声播联网。

3. **【F 簇·方向反转】触屏"不灵敏要重按"**。之前所有触摸工作（含本会话 F1 中值）都在"抗乱跳=更严的门+更多滤波"方向上加码，**与现场主诉"该认的没认"完全相反**。真根因是 `lvgl_press_confirmed()` 的 `sample_count>=2` 恒严门（`axs5106l_touch.c:538-543`），与 4G 无关、WiFi/4G 两批次共有。

4. **【A 簇·已坐实高置信】4G 弱网下字幕卡顿/错误脸**。主事件循环同步跑 SendAudio 阻塞达 10s（`application.cc:372-385` + `ml307_tcp.cc:205`），把字幕/表情 Schedule 饿死。网络 IO 没脱离主任务是显示体验头号杀手。

5. **【B 簇·端侧确定性根因】TTS 应答但无声**。百度协议首批 TTS 帧在状态切到 Speaking 前到达被静默丢弃（`application.cc:690-693`），叠加 `ResetDecoder` 二次清队列（`:1235`），弱网必现"有字幕无声"。

6. **【D 簇】唤醒/打断/误识别**。D1 通话期打断不了=cf0f1672 回退重开了通话唤醒但 SR-AFE 每次重建、AEC 从零收敛漏检（与 0bc897af 内存红线决定冲突）；D2/D3 是同一 wakenet 阈值（0.6）两端取舍；D5 音色切换纯后端。

**全局图景**：18 条反馈里**真正"半夜伤人 + 伤电芯"的是 E+G 簇**，其余是体验退化。E+G 与 C1 互相放大（重启→回 80%→大声播联网）。A/B 同属 4G 弱网 family。F 是历史修复方向性错误。

## 四、F1 触屏裁决（f1TouchVerdict）

**现场主诉是"不灵敏 / 要重按 / 钝"，不是"乱跳"。本会话却新加了压乱跳的 3 点中值滤波——方向拧反了。**

**裁决：中值默认关（`AXS_MEDIAN_FILTER 0`，宏保留待乱跳真机坐实再开），但它不是钝的主因，不要把账算在它头上。**

- **中值该关**：理由不是它造成钝，而是①它服务的目标（压乱跳）与现场诉求（嫌钝）相反；②乱跳尚未真机坐实是高频问题，现场嫌钝是已坐实主诉；③它会让滑动坐标滞后 1-2 帧（空间钝感/拖影）。在"钝"已确诊的当下留着是负收益。证据：`axs5106l_touch.c:728-741`（median 仅 pressed 帧、首帧仅初始化不丢帧）。
- **真根因是阈值（确认门）不是延迟（中值）**：中值 `:737-740` 首帧只初始化历史、不丢帧、不改 sample_count、不增首帧延迟 → 对"认不认这一下"近中性。真正把轻触/快触系统性丢弃的是**多重确认门**：
  - **首恶 = `lvgl_press_confirmed()` 的 `sample_count>=CLICK_MIN_FRAMES(2)`**（`:538-543`,`:48`,`:594-600`）。事件型 INT 一次轻快点击在 33ms 轮询下常只产 1 帧合法读 → sample_count 停在 1 → LVGL 永收不到 PRESSED → Player 暂停/控制中心控件静默丢点。**与 RF 档无关、WiFi/4G 两批次共有**，解释"普遍嫌钝不限 4G"。
  - **4G 更钝叠加项** = `recognize_gesture` 在 RF 活跃（edges>=3）时 `need_frames` 升到 2（`:842-848`），4G 模组发射使 INT 边沿天然密集 → 单帧快点系统性失效。
  - **连点吃掉** = `post_release_guard`（STRICT 300ms，`:578-585`）抬手后 300ms 内二次轻点被整帧吞。
- **结论**：钝是"阈值/确认门"问题，不是"延迟/中值"问题。修中值只是清掉一个错误方向的小负担，真正要动的是 `lvgl_press_confirmed` 的 2 帧门——把"抗乱跳严门"和"要灵敏的 LVGL 控件路径"解耦，不能用一套门服务两个相反目标。**[需真机坐实轻触命中率]**

## 五、修复路线（fixRoadmap，固件/硬件/后端/产品 分流 + 顺序）

### 第一波（固件·1-2 天·性价比最高·停止半夜扰民+丢音量+触屏钝）
1. **E1/E2/G1 — 休眠陀螺仪唤醒治理**（固件，**红线·改电源/休眠需 plan + 真机**）：
   - 方案 A（产品先拍板）：息屏休眠默认 `enable_gyro_wakeup=false`，只保留按键唤醒（`:511` 改 false）。代价=拿起不能自动亮，要按键。
   - 方案 B：保留 gyro 唤醒但加"静止 N 秒 + 显著位移"双确认（复用 SC7A20H，参考 memory `sc7a20h-boot-deadvalue`），过滤桌面微震。
   - 无论哪条：唤醒重启后**先判是否深夜/是否刚联网过**，抑制重复播 `OGG_NETWORK_4G`。**[需 Jack 拍板"拿起自动唤醒"要不要保留 + 真机整夜实测]**
2. **C1 — 删/改 ApplyDefaultSettings 的 `<50` 回写**（固件）：用 `GetInt` 区分"键不存在"（给默认 80）与"键存在但低"（尊重用户），两块板对称改（4g `:857-862` / wifi `:800-805`）。
3. **F1 — 松 LVGL 路径首帧确认**（固件）：`lvgl_press_confirmed` 取消 `sample_count>=2`、仅保留时长门；同时默认关中值。手势 on_click 路径可保留严门。**[需真机轻触命中率]**
4. **C2 — 临时音量不落 NVS**（固件）：闹钟/直播脚本/远程演示走 `SetOutputVolumeTransient`，恢复回 NVS 用户值。

### 第二波（软硬协同·3-5 天·解 4G 弱网 family）
5. **A2/A3/B3 合并治理**（固件为主）：上行音频改独立发送任务 + 有界队列，网络 IO 移出主事件循环（memory `post-prod-stability` 已记断根方向）→ 字幕/表情不再被饿死（A2）、弱网瞬断不升级成哭脸（A3）。同时改 B3：早到 TTS 帧短暂入队暂存不丢、Speaking 入口不无条件 ResetDecoder。
6. **D1 — 通话期打断**（**产品先定方向**）：坚持语音打断 → Speaking 期 SR-AFE 常驻保活（吃内部 RAM，需复核 WiFi 通话 5KB 红线）；回到 0bc897af 决定 → Speaking 关唤醒、打断走按键/触摸并 UI 明示。两条都建议给 AbortSpeaking 加本地即时熔断。**[需 Jack 拍板 + 真机实测命中率]**

### 第三波（产品取舍 + 后端 + 硬件·并行）
7. **D2/D3 阈值**（产品取舍 + 固件）：0.55~0.7 区间真机 ROC 找平衡点；根治需乐鑫定制 wakenet 模型（新功能走 soul-check）。
8. **后端**：B2 曲库版权配置、D5 音色名映射 + ASR 模糊匹配。固件无可改点。
9. **硬件**：C3 响度一致性 / B4 电流声 / A4+F1 触摸 RF 注入根治 / A1 新机闪红屏来料——需示波器/声压计/产线校准。固件侧先统一 box_audio_codec 0.95 缩放（`:194` vs `:249` 两路径差 5%）便于隔离硬件因素。
10. **E4 待坐实**：复核 4g `:254 kHoldRequiredMs=1500`（开机 1.5s）vs 长按关机 3s（`:813`）——现场反馈"开机要 5-6s、关机 3-4s 疑似反了"，需真机量实际时长确认是逻辑反了还是体感误差。**[待现场坐实]**
