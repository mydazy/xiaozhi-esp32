## 🔴 活跃 BUG（不是包袱，会出事故）— 优先修

**git MCP 硬编码 `--repository=/Users/jack/GitHub/mydazy-p30-v2`**，当前工作仓是 v31。任何 `mcp__git__*` 调用会**静默读写旧仓 v2**，造成"以为提交 v31 实际动了 v2"的事故。
- 修法：删除 `~/.claude.json` 的 `mcpServers.git` 条目，git 全回退原生 `Bash(git)`（跟随 cwd，零硬编码；现有 17 条 deny 已覆盖 force-push/reset --hard/clean -fd）。
- 在修掉前：我一律用原生 `Bash(git)`，绝不调 git MCP。

## 三、补充的现代理念规则（草案，可直接采用）

> 这些是"让后续高质量开发更高效"的新增规则。确认后并入对应文件。

### 草案 A — 补进全局 `~/.claude/CLAUDE.md`

```
## 配置纪律（元规则·防常驻层再膨胀）
- rules/ 每份必须有 paths/globs 作用域；常驻层（无作用域）总行数 ≤40 行。
- 凡能被 skill description 自动触发判断的，不写进 rules——不再维护任何"触发词→skill"映射表。
- 发版/部署/OTA 类 skill（ship/land-and-deploy/rollback-plan/setup-deploy）加
  disable-model-invocation:true，只允许显式 /调用；其余靠 description 自动触发。

## 记忆纪律（单一权威链路）
- 项目事实/调试根因/决策 → 写当前项目 memory/*.md（原生 Auto Memory）。跨会话只信这一处。
- 不再用 qdrant / claude-mem / knowledge-graph memory-MCP。memory-repo 仅作 git 跨机备份镜像。
- 分层：CLAUDE.md=must-follow 规则；MEMORY.md=纯索引 ≤150 行；topic .md=具体事实（按需加载）。
  sc7a20h、s31 两个 topic 即正确范式。
- 季度/大版本发版前审查 memory，删过时/已修复 topic。

## 工作纪律（从 auto-skill-dispatch 救出的高信号部分）
- soul-check 触发：用户提"做一个X/新项目/重构/有个想法"且工作量>2h 先拷问。
  跳过：修 bug / 已确认续做 / 查询 / <30min / 明确说"直接做"。
- Jack 行为模式：①想法发散→拉回 ②新不如旧→新技术先 /tech-radar ③做一半停→主动追进度。
  方向确认后执行力强，关键是帮他确认方向。
```

## 四、lean 目标态

- **记忆层**：留 原生 Auto Memory + memory-repo(git 备份镜像)；停 memory MCP/qdrant 6 库/claude-mem；memory-service 按节奏退役。**唯一权威链路。**
- **MCP 层**：全局常驻 11→2（filesystem+context7）；git 回原生 Bash；browser 挪前端项目；云连接器走 ToolSearch 按需。
- **权限层**：bypassPermissions→acceptEdits；留 17 deny；删 4 幽灵 allow；红线文件叠 plan mode。
- **规则层**：全 path-scoped；常驻 166→≤40 行；design 合并；auto-skill-dispatch 瘦身。
- **skill/插件**：ESP32 相关约 10 个 + 原生 11 subagent 作唯一编排底座；弃僵尸 skill；gstack 全家桶仅 Web 项目；副作用 skill 加 disable-model-invocation。
- **hook**：6→3 精准点；删假编译；PreCompact 瘦身；sync 去重单触发点。

**预期净效**：ESP32 量产会话初始化 token 降 20-30% · 磁盘释放 ≥523M · 消除 git 写错仓 bug · 单一记忆事实源 · 权限可见可控且符合量产纪律。
