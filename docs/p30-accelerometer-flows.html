1 概览

SC7A20H 低功耗三轴加速度计 · IIC 经 i2c_bus_worker 共线 · 三大功能：摇一摇互动 / 深睡 EXT1 拿起唤醒 / 闹钟摇晃关停。驱动 v4.0 C handle API。

4 拿起唤醒

4.1 深睡前 arm 流程

P0 时序 bug：当前 EnterDeepSleep 顺序为「arm_wakeup(593) → ShutdownTouchAndAudio(604) → ConfigureDeepSleepWakeupSources(605)」· arm 后到真正 enter sleep 之间窗口 ~500ms（含 200ms vTaskDelay）· 期间任何强动作会把 INT1 拉低 → 立即唤醒。修复：arm_wakeup 挪到 vTaskDelay 200 之后、esp_deep_sleep_start 之前最小化窗口。

4.2 唤醒路径

事件响应
用户拿起设备 · INT1 拉低EXT1_ANY_LOW 触发 · ESP32 唤醒
HandleWakeupCausecase ESP_SLEEP_WAKEUP_EXT1 · 静默 init · first_boot=true
WelcomeTask正常 init · 不放欢迎音（拿起场景静默）

5 闹钟摇停集成（A1）

5.1 当前实现

步动作位置
1OnShake 触发（同摇一摇路径）shake_task → Schedule
2主线程 TryStopAlarmRinger("shake")4g:339 / wifi:343
3响铃中真 → AlarmRinger.Stop · OnShake return—
4非响铃 → 走常规摇一摇互动—

5.2 ⚠ 设计与实现差距

A1 设计阈值切换未实现：响铃中应该切到 800mg / 200ms 灵敏模式（被子翻动可触发），离响铃恢复 1500mg/600ms（防误触）。但当前代码硬编码 1500mg 偏离 · 响铃中只有重摇才能停。
修复方向：① 新增 sc7a20h_set_shake_params(thresh_mg, window_ms) API ② AlarmRinger.Start / Stop 调用切换 ③ 响铃 = 高灵敏 · 默认 = 防误触

6 寄存器配置

init 序列（原子 lock_session 200ms 锁）

寄存器值含义
CTRL1=0x200x00power-down · 配置前必先 down
CTRL4=0x230x98±4g · BDU · HR 高分辨率
CTRL2=0x210x01HPF 路由到 INT1
CTRL5=0x240x08latch INT1（必须读 SRC 才清）
CTRL3=0x220x40AOI1 → INT1（OR 触发模式）
CTRL6=0x250x02INT 极性 active LOW
INT1_THS=0x3210（320mg/32mg LSB）拿起阈值
INT1_DUR=0x3310（100ms/10ms LSB）触发持续时间
INT1_CFG=0x300x2AXHIE|YHIE|ZHIE · 三轴任一超阈即触
CTRL1=0x200x57最后 power-up · ODR 100Hz · XYZ enable

清 latch 序列（arm_wakeup 内）

  • 读 INT1_SRC=0x31（一字节），读完即清 latch
  • 不读 → INT1 仍保持上次低电平 → enable_ext1_wakeup_io 立即认为有事件 → esp_deep_sleep_start 瞬间唤醒
  • v2.1 修复此 bug（原 v1 缺读 SRC 步骤）
