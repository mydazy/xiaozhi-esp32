3 WiFi 信号采集 → UI 链路

WiFi 阈值速查

RSSI 范围icon含义
≥ -60 dBmFONT_AWESOME_WIFI（满格）强信号
-60 ~ -70 dBmFONT_AWESOME_WIFI_FAIR一般
< -70 dBmFONT_AWESOME_WIFI_WEAK弱
未连接 / rssi=-127FONT_AWESOME_WIFI_SLASH无连接
wifi_config_mode_=trueFONT_AWESOME_WIFI 误导显示满格但实际是 AP 配网模式
WiFi 路径优点：esp_wifi_sta_get_ap_info 读 LWIP 内存缓存，μs 级完成，主循环零阻塞。驱动每收一个 beacon (~100ms) 自动更新 ap_record，数据足够新鲜。

4 4G 信号采集 → UI 链路

4G CSQ 阈值速查

CSQ近似 RSSIicon含义
0-9≤ -94 dBmSIGNAL_WEAK弱
10-14-93 ~ -85SIGNAL_FAIR一般
15-19-84 ~ -77SIGNAL_GOOD好
20-31≥ -76SIGNAL_STRONG强
-1 / 99未知SIGNAL_OFF无注册 / modem 未就绪
P0 风险 · 主循环 100ms 阻塞
GetCsq 在 main_app (Core 0 · P10) 主循环线程发 AT，等 URC 回填最多 100ms。期间 Schedule lambda / 闹钟检查 / 状态机事件全部 deferred。
缓解：HTTP binary mode（mp3 播放期间）跳过 AT 直接返缓存。
建议修复：改异步——ml307_net 后台任务定时刷 csq_，UpdateStatusBar 只读缓存不发 AT。
P0 风险 · csq_ 无 staleness 时间戳
binary mode 期间 csq_ 一直返缓存。5min mp3 播放期间用户进电梯 → 失联，icon 仍显示满格 → mp3 结束才发现。
建议修复：csq_ 加 last_csq_us，超 30s 视作 stale 强制返回 SIGNAL_OFF。

6 device_state 白名单

UpdateStatusBar 在 lvgl_display.cc:210 用白名单守护查询 network。其余状态保留上次 icon。

DeviceState查 network？原因
kDeviceStateIdle✓常规态
kDeviceStateStarting✓开机过程，需要尽早显示信号
kDeviceStateWifiConfiguring✓配网态
kDeviceStateListening✓听用户语音，期间用户可能走动
kDeviceStateActivating✓激活态
kDeviceStateSpeaking✗避免 4G UART 与 binary mode 冲突（OTA 升级）
kDeviceStateConnecting✗1-3s 短态，跳过影响小
kDeviceStateUpgrading✗OTA 升级中 · UART 资源被占用
其余 4 态✗FatalError / AudioTesting 等过渡态
设计意图：5 个白名单态都是"用户能看到屏幕 + 不阻塞关键 UART"的状态。Speaking / Upgrading 跳过查询避免 AT 与音频/OTA 流冲突。

7 NetworkEvent 事件清单

事件来源application.cc 响应UI 表现
ScanningWiFi 扫描开始ShowNotification "扫描中" 30s顶栏通知
ConnectingWiFi / 4G 注册中SetStatus / ShowNotification 30s状态栏文字
ConnectedWiFi 拿到 IP / 4G 注册成功ShowNotification + setBits(NETWORK_CONNECTED)顶栏 icon 立即刷新
DisconnectedWiFi 断 / 4G 注销setBits(NETWORK_DISCONNECTED)顶栏 icon 立即变 SLASH/OFF
WifiConfigModeEnterWifiBoard 内部—WifiBoard 自己显示 QR
WifiConfigModeExit同上—退出 QR
ModemDetectingML307 模组检测SetStatus "检测模组"状态文字
ModemErrorNoSim无 SIMAlert 弹窗 + 音效错误页
ModemErrorRegDenied注册被拒Alert + 音效错误页
ModemErrorInitFailedmodem 初始化失败Alert + 音效错误页
ModemErrorTimeout注册超时SetStatus "注册中"状态文字
关键缺口：NetworkEvent 枚举不包含 信号强度档位变化 事件。RSSI -65 → -75（跨阈值）需等下一个 10s 周期才反映到 icon。如需精细化建议引入 SignalChanged 事件，但当前实现已满足量产。
