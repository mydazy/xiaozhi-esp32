3.1 WebSocket 接入参数

合规性差距: ptime 帧长协商
官方规范是 URL 参数 ?ptime=60 (RFC 风格), 而 P30 v2.2.12 是在 SendInitialDeviceInfo 里塞 audio_frame_duration_ms / tts_frame_duration_ms / frame_duration_ms 三个候选字段名 — 当时是反推服务端字段名。
影响: 当前 v2.2.13 已退回静态 24000/20 ms; 若未来要重启 60ms 上行, 应该改成 URL 参数 &ptime=60 + &plen=..., 与服务端规范对齐, 不再依赖 DeviceInfo 协商字段名猜测.
预期收益: 帧长协商 100% 成功 (规范明确); 减少弱网上行 packet 数 3 倍 (20ms → 60ms).

8 教育卡 P0 方案 = TAG_MSG 标签消息

P30 教育卡 (中英文识字图 + AI 释义) 的百度官方落地路径已经存在, 不需要自创协议. 完整方案如下:

Step 1 · 控制台配置 (服务端)

  • 创建互动应用, 添加"自定义角色"
  • 角色 prompt 中约定 ((CUSTOM:json)) 标签格式 + 内容 schema
  • 将自定义角色设为默认
#你是一位资深的早教老师，你每次会收到物品的中文名字，
#输出它的中文汉字和英文单词名称及如字典一样详细的中英文释义。
##参考以下示例，如你收到物品名字"花"，按同样格式输出，
##注意((CUSTOM:...))为标签化内容必须包含，内容为json格式：

花。
flower。
((CUSTOM:
{
  "name_ch": "花",
  "name_en": "flower",
  "comments_ch": {
    "name": "花",
    "strokenumber": 7,
    "Radical": "艹",
    "comment": "种子植物的有性繁殖器官...",
    "example_sentence": "学校里种了许许多多五颜六色的~。"
  },
  "comments_en": { ... }
}
))

###按以下内容来应答：
  汉字：{words}。
  单词：{words}。
  ((CUSTOM:{...同上结构...}))

Step 2 · 服务端处理 (BRTC 平台自动)

  • 模型输出文本经服务端正则拆分: 非标签内容 → TTS 播报队列; ((CUSTOM:json)) 标签 → 单独打包
  • 标签作为下行事件: [E]:[TAG_MSG]:"((CUSTOM:{name_ch:"水杯",name_en:"water cup",...}))"

Step 3 · P30 客户端 (本仓库需新增)

  • WebsocketBaiduProtocol: HandleEvent 增加 [E]:[TAG_MSG]: 分支, 解析出 ((CUSTOM:...)) 字符串
  • 提取标签内 JSON, 转发到 EduCard Scene (新场景, 已在 SceneType 落位表占位 P0)
  • EduCard Scene 渲染: 汉字 (大字) + 笔画数 + 部首 + 拼音/英文 + 释义 + 例句, 可配 GIF 笔画动画
关键优势: 这套方案不需要新协议 / 不动状态机 / 不抽基类. 只用已有 Scene 抽象 + 1 个新事件分支就能落地. 完全符合 P30 量产纪律 (review § 一.5.0 第一性原理).

依赖: ① 服务端控制台 prompt 模版 (协同 UserServer 团队); ② P30 端 UiDisplay 新增 EduCard scene (本仓库工作 4-6 周).

9 云端音频 3A 调优 (cloud_3A_url)

P30 目前 100% 依赖端侧 AFE (ESP_SR), 适合中高端机型 (Korvo-2L). 弱机型 (单麦 + 廉价 codec) 可借云端 3A 兜底.

9.1 启用入口 (UserServer 配置)

// generateAIAgentCall 创建智能体时:
{
  "config": {
    "cloud_3A_url": {
      "AEC": { "enable": true, "enhanced": true, "audiobm": 100 },
      "ANS": { "enable": true, "preMode": "L", "midGainDb": 0, "dfLimitDb": 75 },
      "AGC": { "enable": true, "maxVolume": 60, "extraGain": 0 }
    }
  }
}

9.2 默认值

功能默认P30 建议
AEC (回声消除)禁用P30 端侧 ESP-SR AEC 已开 → 云端关; P30 弱机型 (无端侧 AEC) → 云端 enable:true
ANS (噪声抑制)启用保持启用; 嘈杂环境可调 preMode=H (高强度)
AGC (自动增益)禁用P30 已 es7210_adc_set_gain 端侧调过; 量产个体差异大 → 启用兜底 maxVolume=60

9.3 不要同时开端 + 云 3A

端侧 + 云端 3A 同时启用会两次降噪 / 两次回声消除 / 两次增益, 出现"机器人音 / 削顶失真 / 音量不稳"等问题. 必须二选一. 默认 P30 用端侧, 仅弱机型批次切云端.
