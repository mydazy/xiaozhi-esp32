## 二、脏帧守卫审查（红线）✅ 正确

4G 弱网下行脏帧（09-P0 系列）的守卫，逐项核对均"先校验长度再使用"：

| 位置 | 守卫 | 结论 |
|------|------|------|
| `mqtt_protocol.cc:257` | `if (data.size() < sizeof(aes_nonce_)) { LOGE; return; }`，**之后**才 `decrypted_size = data.size() - aes_nonce_.size()` | ✅ 先校验后相减，无 size_t 下溢 |
| `mqtt_protocol.cc:261` | `if (data[0] != 0x01) return;` 包类型校验 | ✅ |
| `mqtt_protocol.cc:267/268` | `memcpy(&ts_raw, &data[8], 4)` 用 memcpy 避免未对齐读 | ✅ 越界已被 257 守卫前置拦截 |
| `websocket_protocol.cc:116/137` | `if (len < sizeof(BinaryProtocol2/3)) ...` | ✅ 二进制协议长度守卫 |
| vendor `78__esp-ml307` | commit `0add7ac24` CME ERROR 空 arguments 越界守卫（09-P0-D）；`723244027` UART 溢出假断网恢复 + 16K 接收缓冲 | ✅ 4G 稳定性修复 |

**全部默认保留，不动。**

## 三、板级改动判定（ml307_board / dual_network）

| 改动 | 官方 v2.2.4 | 为什么 | 判定 |
|------|-----------|--------|------|
| NetworkTask 开头 `vTaskDelay(8000ms)` | 无延迟 | ML307R VBAT 上电后 UART Active 需等待，不等则 AT 通信失败 | 🟢 硬件时序（⚠️见第四节注释瑕疵）|
| `GetBoardJson` 改用 `GetCsqCached/GetCarrierNameCached/GetRegistrationStateCached` | 调 `GetCsq()` 等发 AT 命令阻塞 | OTA 上报 board json 不被 AT 命令阻塞（弱网/modem 忙时） | 🟢 **防阻塞**（对应 memory 弱网卡死 P1）|
| ml307_net 任务 `xTaskCreatePinnedToCore(core 0)` | 不绑核 | 4G 在 core0、音频在 core1 隔离（对应 4G-RF 专题）| 🟢 核隔离 |
| `modem_==nullptr` 守卫（GetBoardJson/GetDeviceStatusJson）| 直接解引用 | modem 未初始化时返回最简 json | 🟢 空指针防护 |
| CSQ 分档阈值统一（0-14 weak / 15-19 fair / 20-24 good / 25-31 strong）| 两处分档不一致 | GetNetworkStateIcon 与 GetDeviceStatusJson 分档对齐 | 🟢 一致性小改 |
| `GetModem()` 暴露接口 | 无 | GNSS 扩展 + network_ready 检查 | 🟢 有调用方（`mydazy_p30_board.cc:566/1090`，非死接口）|
| dual_network 切网加 `PlaySound(OGG_NETWORK_4G/WIFI)` | 仅 ShowNotification | 切网语音反馈 | ⚪ 体验扩展 |
| dual_network `CanSwitchNetwork()=true` + `SwitchNetwork()` override | 无 | 暴露给控制中心切网入口 | 🟢 双网功能配套 |

**飞行模式省电**（CFUN=4 优雅释放 PPP，`mydazy_p30_board.cc` GracefulShutdownModem）：深睡前优雅断 4G 而非硬断，避免脏断导致下次注册变慢 = 🟢 标准做法（详细编排归 board 模块）。

## 四、⚠️ 轻微观察（非过度，建议核对）

### 8s 延迟 vs 注释"≥10s"不一致

- **位置**：`ml307_board.cc:68` 注释写"VBAT 上电后 UART Active 需 ≥ 10s"，实际 `vTaskDelay(8000ms)` 只延 8s。
- **性质**：注释与实现差 2s。可能：①别处还有等待凑够 10s ②实测 8s 已够。
- **风险**：若 8s 不足，弱批次 ML307R 可能偶发上电 AT 超时（开机变慢/首次注册失败重试）。**属硬件时序，不是过度优化**。
- **建议**：核对实测依据，注释与实现取齐（要么改注释说明 8s 实测足够，要么补足到 10s）。**不计入 🔴**。
