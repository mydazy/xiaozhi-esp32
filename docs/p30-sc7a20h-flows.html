1 模块简介

SC7A20H 是 Silan Microelectronics 推出的低功耗三轴加速度计 · 寄存器布局与 ST LIS2DH12/LIS3DH 完全兼容 · 本驱动针对 MyDazy P30 量产场景做极简封装 · 仅暴露刚需 API · 全部 I/O 走 i2c_bus_worker 防 4G RF 共线污染。

1.4 设计哲学

  • 极简 API：3 个 API 覆盖全部产品场景（v5.0 规划）· 不开放运行时切量程/ODR/del
  • 量产保守：所有阈值默认值经实测调优 · 调校仅改源码 · 不开放 OTA 调参
  • 共线保护：I²C 全部走 i2c_bus_worker（与 codec ES8311 / 触摸 AXS5106L 共线）· init 10 寄存器序列走 lock_session 200ms 原子锁
  • 失败优雅：I²C 失败不阻塞唤醒注册（LOGW 警告 + 继续）· LIR_INT1=0 防 latch 残留
  • 双核纪律：motion_task 在 Core 1 P1 优先级 · 与 LVGL 共核但低于 LVGL P5

2 应用场景

本驱动仅服务三个产品场景 · 每个场景一段触发链路。

2.1 场景 ① 拿起唤醒（深睡 → 醒）

目的设备深睡中，用户拿起设备 → ESP32 唤醒并进入正常工作流
触发条件三轴任一加速度偏离 ≥ 320 mg 持续 ≥ 100 ms
实现SC7A20H 硬件 AOI1 → INT1 拉低 → ESP32 EXT1_ANY_LOW 唤醒
触发频率低（设备深睡才生效 · 活跃态此中断不参与业务）
用户控制NVS Settings("status").pickupWake 总开关 · 远程命令 {"type":"sleep","gyro":bool} 单次禁用

2.2 场景 ② 摇一摇互动（活跃态 → AI 对话）

目的儿童拿起设备甩动 → 触发"摇一摇随机互动" SendTextToAI
触发条件600 ms 滑窗内 ≥ 2 帧"偏离 1g 模长平方 ≥ 1500² mg²"（上下 2 次即触发）
实现软件 motion_task 100 ms 周期采样 · 整数模长平方算法 · 1500 ms 触发冷却
闹钟摇停响铃中 `AlarmRinger::ShakeStop(3)` 累计 5s 窗内 3 次摇晃才停（防走路误关 · 详见 § 9）
用户控制4 个平铺参数（deviation_mg / window_ms / target_frames / cooldown_ms）调用时传入

2.3 场景 ③ 桌面双击（活跃态 → 唤醒屏幕）v5.0 已实现 · 当前暂关

目的用户敲击桌面两次 → 调 WakeUp() 唤醒屏幕（与触摸唤醒同语义 · 不进 AI 对话）
触发条件两次单帧瞬态峰值 ≥ 1800 mg · 间隔在 [80, 400] ms · 中间存在归零静止帧
实现软件 motion_task 内状态机（IDLE → WAIT_GAP → TRIGGER）· 与 shake 共享 100ms 采样数据
状态驱动 sc7a20h_strike 已实现 · 三板 board.cc 调用已 注释暂关 · 后续真机调校验证后启用
用户控制4 个平铺参数 · P31 整机重量 ~60g → peak_mg 调高至 2200（防误触）

5 API 详细说明

v5.0 共 4 个公开 API · 所有灵敏度参数在调用时直接传入（不用 cfg struct）· 调用顺序：init 一次 → shake/strike 各一次（任选） → wakeup 每次进深睡。

5.1 sc7a20h_init — 基础初始化 + 拿起灵敏度

sc7a20h_handle_t sc7a20h_init(i2c_worker_handle_t worker,
                               uint16_t pickup_thresh_mg,
                               uint16_t pickup_duration_ms);

注册到 i2c_worker · WHO_AM_I 校验 · 10 寄存器原子序列初始化 · 配置硬件 AOI1 拿起检测。示例：sc7a20h_init(i2c_worker_, 320, 100);

参数说明
worker已创建的 i2c_bus_worker 句柄 · 必须共享 codec/touch 总线
pickup_thresh_mg拿起阈值 mg · 步长 32 · 三板项目值 320
pickup_duration_ms拿起持续 ms · 步长 10 · 三板项目值 100
返回含义
非 NULL handle初始化成功 · 后续 API 用此句柄
NULLworker 为空 / calloc 失败 / I²C 设备添加失败 / WHO_AM_I ≠ 0x11 · 自动清理资源

5.2 sc7a20h_wakeup — 深睡前 arm

esp_err_t sc7a20h_wakeup(sc7a20h_handle_t h, gpio_num_t int1_gpio);

清 INT1 残留 latch（兜底）+ 注册 EXT1 ANY_LOW wakeup + RTC GPIO 上拉。示例：sc7a20h_wakeup(h, GPIO_NUM_3);

  • 调用时机：EnterDeepSleep 中 · 每次进深睡都调用 · 必须在 audio shutdown 之前（详见 docs § 8.4）
  • I²C 失败兜底：读 INT1_SRC 失败仅 LOGW · 不阻塞 EXT1 注册（LIR=0 保证 INT1 不会被钉住）
  • RTC 配置：函数内完成 rtc_gpio_pullup_en + rtc_gpio_pulldown_dis · 上层无需再配 GPIO
  • 返回：ESP_OK / ESP_ERR_INVALID_ARG（h=NULL）/ esp_sleep_enable_ext1_wakeup_io 透传错误

5.3 sc7a20h_shake — 摇一摇检测

esp_err_t sc7a20h_shake(sc7a20h_handle_t h,
                             uint16_t deviation_mg, uint16_t window_ms,
                             uint8_t target_frames, uint16_t cooldown_ms,
                             sc7a20h_shake_cb_t cb, void *ctx);

启动摇一摇检测 · 首次调用启动共享 motion_task（栈 2560 · P1 · Core 1）。示例：sc7a20h_shake(h, 1500, 600, 2, 1500, &OnShake, this);

参数项目默认说明
deviation_mg1500偏离 1g 阈值 mg · 内部平方判定
window_ms600滑窗时长 · 须 100 整数倍 · 上限 1600（16 帧）
target_frames2窗内强动帧触发阈值 · "上下 2 次即触发"
cooldown_ms1500触发后冷却 ms · 防连发
cb—触发回调（不可 NULL）· 签名 void(void *ctx)
  • 返回：ESP_OK / INVALID_ARG（h/cb=NULL）/ INVALID_STATE（重复启动）/ NO_MEM（task 创建失败）
  • cb 上下文：motion_task · 非 ISR · 栈仅 2560B · 不可同步阻塞 · 必须 Application::Schedule
  • 闹钟摇停：cb 内调 AlarmRinger::ShakeStop(3) 累计 5s 窗 3 次才停（防走路误关）

5.4 sc7a20h_strike — 桌面双击检测

esp_err_t sc7a20h_strike(sc7a20h_handle_t h,
                               uint16_t peak_mg, uint16_t min_gap_ms,
                               uint16_t max_gap_ms, uint16_t cooldown_ms,
                               sc7a20h_strike_cb_t cb, void *ctx);

启用桌面双击检测 · 与 shake 共享 motion_task。触发后业务是"唤醒屏幕"（板级 cb 调 WakeUp · 不进 AI 对话）。示例：sc7a20h_strike(h, 1800, 80, 400, 800, &OnStrike, this);

参数项目默认说明
peak_mg1800（P31 调 2200）单次撞击瞬态峰值阈值 mg
min_gap_ms80两击最小间隔 ms · 防抖下限
max_gap_ms400两击最大间隔 ms · 双击窗口上限
cooldown_ms800触发后冷却 ms
cb—触发回调（不可 NULL）· 签名 void(void *ctx)
  • 返回：同 sc7a20h_shake
  • 状态机：IDLE → WAIT_GAP（检到第一击 peak）→ TRIGGER（检到第二击 + gap 在 [min, max] 区间）→ COOLDOWN
  • cb 业务建议：调板级 WakeUp()（与触摸唤醒同语义）· 不要 SendTextToAI

8.1 寄存器地图

寄存器地址方向作用
WHO_AM_I0x0FR固定值 0x11
CTRL_REG10x20R/WODR + XYZ axis enable
CTRL_REG20x21R/WHPF 路由
CTRL_REG30x22R/WINT1 路由
CTRL_REG40x23R/W量程 + BDU + HR

8.2 init 序列（原子 lock_session 200ms 锁）

步寄存器值位含义
1CTRL10x00power-down（配置前必先 down）
2CTRL40x98±4g + BDU + HR 12-bit（0x10 \| 0x88）
3CTRL20x01HPF 路由到 INT1
4CTRL50x00LIR_INT1=0 · INT1 不锁存（v4.0.1）
5CTRL30x40I1_AOI1 → INT1（OR 触发模式）
6CTRL60x02H_LACTIVE=1 · INT 极性 active LOW
7INT1_THSths（用户传 320 → 10）拿起阈值 = ths × 32 mg
8INT1_DURdur（用户传 100 → 10）拿起持续 = dur × 10 ms
9INT1_CFG0x2AXHIE \| YHIE \| ZHIE · 三轴任一高侧超阈触发
10CTRL10x57ODR 100Hz + XYZ enable（0x50 \| 0x07）

8.3 读数据序列

  • burst read OUT_X_L · 寄存器地址 MSB=1（0x28 \| 0x80 = 0xA8）使 sensor 自动递增
  • 一次读 6 字节 = X_L X_H Y_L Y_H Z_L Z_H
  • 12-bit 数据 left-aligned · 算术右移 4 还原 signed 12-bit · 乘 2（mg/LSB）得 mg

8.4 LIR_INT1=0 关键修复（v4.0.1 · 2026-05-12）

原 bug：CTRL5=0x08（LIR=1 锁存）· 配合"读 INT1_SRC 清 latch"模式 · 在 audio shutdown 之后调 arm_wakeup 时 codec ES8311/ES7210 失电 → 通过 ESD 二极管短路 SDA/SCL → i2c_master_transmit_receive INVALID_STATE → INT1_SRC 清不掉 → INT1 被旧 latch 钉 LOW → esp_deep_sleep_start 瞬间唤醒。
修复：① CTRL5=0x00（LIR=0 不锁存）· INT1 跟随运动动态电平 · 静止 100ms 后自动归 HIGH · 完全不依赖 SRC 清除。② arm_wakeup 反向挪回 audio shutdown 之前（board:594）· I²C 仍健康时读 INT1_SRC 兜底清残留 latch（即使失败 LIR=0 也保证 INT1 自动归 HIGH）。双保险锁死秒醒根因。

8.5 arm_wakeup INT1_SRC 兜底清 latch

  • 读 INT1_SRC（0x31）一字节 · 寄存器读即清
  • v4.0 时是主防线（latch=1 必须清才能解锁）· v4.0.1 LIR=0 后退化为 defense-in-depth 兜底
  • 读失败：仅 ESP_LOGW · 不阻塞 EXT1 注册 · LIR=0 已保证主路径安全

    /* ... ResetProtocol / ShutdownModem ... */

9.3 闹钟摇停（AlarmRinger 内置累计）

/* main/audio/alarm_ringer.cc */
bool AlarmRinger::ShakeStop(int min_count) {
    if (!IsRinging()) return false;
    int64_t now = esp_timer_get_time();
    if (now - shake_first_us_ > 5000000LL) {     // 5s 滑窗
        shake_count_ = 0;
        shake_first_us_ = now;
    }
    if (++shake_count_ >= min_count) {
        Stop("shake×N");
        shake_count_ = 0;
    } else {
        ESP_LOGI(TAG, "shake stop pending (%d/%d)", shake_count_, min_count);
    }
    return true;
}

11 注意事项

11.1 I²C 总线纪律

  • SC7A20H 与 ES8311 codec / AXS5106L touch / 4G modem AT 共享 I²C_NUM_1
  • 所有事务必须走 i2c_bus_worker · 不可裸调 i2c_master_transmit · 否则 4G RF 干扰下数据错
  • init 10 寄存器序列必须用 i2c_worker_lock_session(worker, 200) 原子包裹 · 防 ISR 中途插入
  • 从 ISR / 中断回调中禁止发起 I²C 事务（worker queue 是 FreeRTOS 队列）

11.2 arm_wakeup 时序铁律

必须严格在 audio shutdown 之前调用。AUDIO_PWR_EN=0 后 codec 失电 · ESD 二极管短路 SDA/SCL · 此时 I²C 事务返回 ESP_ERR_INVALID_STATE · INT1_SRC 清不掉 → 历史 bug "睡眠秒醒"（已修 · v4.0.1 LIR=0 双保险）。
  • 正确顺序：arm_wakeup → ShutdownTouchAndAudio → ConfigureDeepSleepWakeupSources → vTaskDelay(200) → esp_deep_sleep_start
  • vTaskDelay(200) 必须保留 · 等 AUDIO_PWR_EN=0 引发的机械瞬态衰减（LIR=0 配合让 INT1 自动归 HIGH）

11.3 量产保守原则

  • 不开放运行时切量程/ODR · 改值需改源码 + 重 init 全流程
  • 不开放运行时切灵敏度 · A1 闹钟"800mg 灵敏切换"如真需要应另起 set_xxx_params API · 量产期不做
  • 所有阈值三板同步改 · 量产期"统一性 > 灵活性"
  • 改默认值必须真机回归：日常 / 走路 / 桌面 / 儿童使用 · 24h 误触率

11.4 双核与栈纪律

  • motion_task 必须 Pin Core 1（与 LVGL/AFE 同核 · 但低优先级 P1 不抢占）
  • 栈 2560B 仅够整数算术 + I²C burst read · cb 不可同步阻塞（必须 Schedule）
  • 禁止 PSRAM 栈 + Core 0 组合（cache 共 SPI · flash op 期间 PSRAM 栈失活 · 见 CLAUDE.md 内存红线）
  • 所有阻塞 API 必须超时（实时 100ms / 普通 500ms / 后台 5s）· 禁 portMAX_DELAY 裸用
